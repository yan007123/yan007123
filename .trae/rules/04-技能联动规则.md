# 技能联动规则

## 核心原则

**用户一句话，系统自动触发完整流程。**

---

## 一、自动触发规则

### 1.1 关键词触发表

| 用户输入 | 自动触发 | 执行流程 |
|---------|---------|---------|
| "写第X章" | 章节创作流程 | 大纲→思考→人物→章节→人味→质检→记忆 |
| "设计人物" | 人物设计流程 | 思考→人物卡→记忆 |
| "调整大纲" | 大纲调整流程 | 思考→大纲管理→记忆 |
| "检查质量" | 质检流程 | 质检SKILL→反馈 |
| "埋设伏笔" | 伏笔管理 | 大纲管理→记忆更新 |
| "查看人物" | 人物查询 | 人物卡SKILL→读取记忆 |
| "查看大纲" | 大纲查询 | 大纲管理→读取记忆 |

### 1.2 触发示例

**用户说**："帮我写第5章"

**系统自动**：
```
1. 加载记忆 → 获取第4章状态、人物位置、伏笔进度
2. 顺序思考 → 规划第5章内容（5步模板）
3. 人物卡 → 确认出场人物设定
4. 章节SKILL → 生成章节内容
5. 人味SKILL → 去AI味处理
6. 质检SKILL → 检查逻辑和一致性
7. 更新记忆 → 记录第5章状态、伏笔变化
```

**用户说**："设计一个新人物，是萧渊的旧部"

**系统自动**：
```
1. 顺序思考 → 人物设计（3步模板）
2. 人物卡 → 创建人物档案
3. 更新记忆 → 记录新人物、关系图谱
```

## 二、标准执行流程

### 2.1 章节创作流程

```yaml
流程名称: 章节创作
触发词: ["写第X章", "创作第X章", "生成第X章"]

步骤:
  1_加载记忆:
    工具: Memory读取
    内容: [项目状态, 人物状态, 伏笔进度, 爽点规划]
    
  2_顺序思考:
    工具: Sequential Thinking
    模板: 章节创作模板(5步)
    输出: [场景设计, 情感节奏, 心声规划, 钩子设计]
    
  3_人物确认:
    工具: 人物卡SKILL
    输入: 出场人物列表
    输出: 人物设定确认
    
  4_生成内容:
    工具: 章节SKILL
    输入: [思考结果, 人物设定]
    输出: 章节正文
    
  5_去AI味:
    工具: 人味SKILL
    输入: 章节正文
    输出: 优化后正文
    
  6_质量检查:
    工具: 质检SKILL
    检查项: [逻辑一致性, 人物OOC, 伏笔呼应, 情感节奏]
    输出: 质检报告
    
  7_更新记忆:
    工具: Memory更新
    内容:
      - 章节完成状态
      - 人物关系变化
      - 伏笔埋设/回收
      - 爽点完成标记

总耗时预估: 3-5分钟
```

### 2.2 人物设计流程

```yaml
流程名称: 人物设计
触发词: ["设计人物", "创建人物", "新人物"]

步骤:
  1_顺序思考:
    工具: Sequential Thinking
    模板: 人物设计模板(3步)
    输出: [基础设定, 深度设定, 应用规划]
    
  2_创建档案:
    工具: 人物卡SKILL
    输入: 思考结果
    输出: 人物档案
    
  3_更新记忆:
    工具: Memory更新
    内容:
      - 新人物加入人物图谱
      - 关系网络更新
      - 出场章节标记

总耗时预估: 1-2分钟
```

### 2.3 大纲调整流程

```yaml
流程名称: 大纲调整
触发词: ["调整大纲", "修改大纲", "重构大纲"]

步骤:
  1_加载记忆:
    工具: Memory读取
    内容: [当前大纲, 已完成章节, 伏笔状态]
    
  2_顺序思考:
    工具: Sequential Thinking
    深度: 深度思考(8-12步)
    输出: [问题诊断, 调整方案, 影响分析]
    
  3_大纲管理:
    工具: 大纲管理SKILL
    输入: 调整方案
    输出: 更新后大纲
    
  4_冲突检测:
    工具: 质检SKILL
    检查: [伏笔冲突, 人物冲突, 逻辑冲突]
    输出: 冲突报告
    
  5_更新记忆:
    工具: Memory更新
    内容:
      - 大纲版本更新
      - 伏笔调整记录
      - 爽点重新规划

总耗时预估: 5-8分钟
```

### 2.4 质检流程

```yaml
流程名称: 质量检查
触发词: ["检查", "质检", "有问题吗", "OOC了吗"]

步骤:
  1_加载标准:
    来源: 05-质检标准规则
    内容: [去AI味标准, 一致性标准, 女频专项标准]
    
  2_执行检查:
    工具: 质检SKILL
    检查项:
      - 句式多样性
      - 词汇离散度
      - 情感具象化
      - 人物一致性
      - 伏笔呼应
      - 五感描写
      - 钩子设计
    
  3_生成报告:
    格式:
      - 通过项 ✅
      - 警告项 ⚠️
      - 错误项 ❌
      - 修改建议

总耗时预估: 1-2分钟
```

## 三、流程中断与恢复

### 3.1 中断场景

| 场景 | 处理方式 |
|------|---------|
| 用户中途打断 | 记录当前步骤，保存到短期记忆 |
| 发现重大冲突 | 暂停流程，询问用户意见 |
| 需要用户确认 | 输出当前结果，等待用户反馈 |
| 工具调用失败 | 重试3次，失败后报告用户 |

### 3.2 恢复机制

```
用户说: "继续"
  ↓
读取短期记忆
  ↓
从中断步骤继续
  ↓
完成剩余流程
```

## 四、反馈机制

### 4.1 每步反馈

**执行每个步骤后，简要告知用户**：
- "已加载项目记忆，当前写到第5章"
- "正在规划第5章内容..."
- "已确认人物设定，萧渊内功受损"
- "正在生成章节正文..."

### 4.2 最终报告

**流程完成后，输出执行报告**：
```
✅ 章节创作完成

执行摘要:
- 章节: 第5章 靠肩而眠
- 字数: 2500字
- 情感标签: 💕 甜章
- 心声: 5处
- 钩子: ★★★★☆

质检结果:
✅ 句式多样性
✅ 词汇离散度
✅ 情感具象化
⚠️ 嗅觉描写较少（建议增加）

已更新:
- 第5章完成状态

- 伏笔V004进展

下一步建议:
- 第6章建议写⚔️冲突章（山间遇险）

```

## 五、快捷命令

### 5.1 单命令触发

| 命令 | 效果 |
|------|------|
| "继续" | 从中断处恢复 |
| "跳过" | 跳过当前步骤（慎用）|
| "重做" | 重新执行当前步骤 |
| "详细" | 输出当前步骤详细过程 |
| "简化" | 使用简要模式执行 |

### 5.2 组合命令

| 命令 | 效果 |
|------|------|
| "写第5章，简化模式" | 跳过详细思考，快速生成 |
| "检查第3章，详细模式" | 详细质检报告 |
| "设计人物，女配，反派" | 直接指定人物类型 |

---

## 六、错误处理

### 6.1 常见错误

| 错误 | 原因 | 处理 |
|------|------|------|
| 记忆读取失败 | Memory工具错误 | 使用本地文件备份 |
| 思考步骤超时 | 任务太复杂 | 拆分为子任务 |
| 人物设定冲突 | 与已有设定矛盾 | 询问用户确认 |
| 质检不通过 | 质量问题 | 自动修复或报告 |

### 6.2 降级策略

```
完整流程失败时:
  1. 尝试简化流程（跳过非关键步骤）
  2. 尝试分步执行（一次执行一个步骤）
  3. 报告用户，说明问题
```
