# Sequential Thinking 顺序思考工具

> 每个阶段必须使用，用于目标解析、任务拆解、分步执行等所有操作。
> 触发场景：复杂问题、多阶段任务、需要顺序执行的任务等全场景。
> 所有交互对话必须执行此工具，不能跳过任何操作。

---

## 核心定义

Sequential Thinking（顺序思考）是一种**结构化推理工具**，通过将复杂问题分解为可管理的步骤来逐步推进思考。

它不是简单的"想一想"，而是：
- 生成假设
- 验证假设
- 根据结果调整方向
- 追踪思维演进历史

---

## 触发规则

### 强制触发场景（以下任一情况必须执行）

| 场景 | 说明 | 执行深度 |
|-----|------|---------|
| **创作任务** | 用户要求写章节/场景/对话 | 完整思考链（5-10步） |
| **规划任务** | 用户要求规划剧情/大纲/人物 | 完整思考链（5-10步） |
| **分析任务** | 用户要求分析问题/设定/关系 | 完整思考链（5-10步） |
| **复杂问题** | 问题涉及多个因素或变量 | 视复杂度定 |
| **决策任务** | 用户需要做选择 | 至少3个选项的利弊分析 |
| **问题诊断** | 排查问题原因 | 至少3层因果追溯 |

### 推荐触发场景（建议执行）

| 场景 | 说明 |
|-----|------|
| 用户提出模糊需求 | 需要先澄清再执行 |
| 可能存在多种解法 | 需要分析对比 |
| 涉及跨文件/跨模块 | 需要协调多个资源 |
| 风险较高 | 需要预判后果 |

## 标准执行流程

### Step 1：问题/任务识别

**第一个思考步骤必须包含：**
```
任务类型：[创作/规划/分析/决策/其他]
核心问题：[用一句话描述要解决的核心问题]
约束条件：[如有明确限制条件]
预期输出：[用户期望的结果形式]
```

### Step 2：分解与假设

```
将复杂问题分解为：
1. 子问题A
2. 子问题B
3. 子问题C

可能的解决路径：
- 路径1：[简述]
- 路径2：[简述]
- 路径3：[简述]
```

### Step 3-7：逐步推理

每个思考步骤需要：
- 基于前一步骤的结论
- 引入相关信息
- 生成新的洞察或结论
- 明确下一步方向

### 最终步骤：总结与行动

```
结论：[核心发现/最终决策]

建议行动：
1. [具体步骤]
2. [具体步骤]
3. [具体步骤]

风险提示：[如有]
```

## 参数说明

| 参数 | 类型 | 说明 |
|-----|------|------|
| thought | string | 当前思考步骤的内容 |
| thoughtNumber | number | 当前步骤编号 |
| totalThoughts | number | 预估的总步骤数 |
| nextThoughtNeeded | boolean | 是否需要继续思考 |
| isRevision | boolean | 是否修正了之前的思考 |
| revisesThought | number | 修正的是哪一步（如适用） |
| branchFromThought | number | 从哪一步分支（如适用） |
| branchId | string | 分支标识（如适用） |
| needsMoreThoughts | boolean | 是否意识到需要更多思考 |

## 执行示例

### 示例场景：用户要求规划第5章

```
Step 1:
任务类型：创作规划
核心问题：如何设计第5章的核心场景和情感节奏
约束条件：延续第4章结尾的"跟随流放"设定
预期输出：5个候选故事梗概供用户选择

Step 2:
分解：
1. 当前状态分析（第4章结尾）
2. 核心矛盾确立（验证心声真假）
3. 场景设计（流放路途）
4. 情感节奏设计（推拉）
5. 章末钩子设计

路径：
- 路径A：侧重路途艰辛（现实主义）
- 路径B：侧重心声互动（甜向）
- 路径C：侧重悬疑暗线（眼线伏笔）
...

[后续步骤继续展开]
```

## 与 Memory 工具的联动

| 阶段 | Sequential Thinking | Memory |
|-----|-------------------|--------|
| 任务开始 | 执行思考链 | 可能需要加载记忆 |
| 任务进行中 | 定期执行思考 | 定期更新短期记忆 |
| 任务完成 | 给出最终结论 | 升级/更新中期记忆 |
| 新任务开始 | 新建思考链 | 检查是否需要加载记忆 |

## 违规处理

如果跳过 Sequential Thinking 工具的执行，将收到系统警告：
> ⚠️ Sequential Thinking 工具未被执行。请使用 mcp_Sequential_Thinking_sequentialthinking 工具进行结构化思考。

---

## 注意事项

1. **不要跳步**：复杂任务至少需要5个以上思考步骤
2. **记录历史**：使用 isRevision 和 revisesThought 追踪思维修正
3. **分支管理**：复杂问题可分支，但确保主路径清晰
4. **提前终止**：如果 3 步内能完全解决，不需要强制走完
5. **保持聚焦**：每一步都应推进主线，避免过度发散
