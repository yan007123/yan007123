# Memory 三层记忆架构

> 每个阶段必须使用，核心目标是将零散、非结构化的用户信息转化为可结构化、可查询、可演化的长期记忆，并在后续对话中被持续维护和利用。

---

## 触发规则

- **所有交互对话必须执行 Memory 工具**，不能跳过任何操作
- 新会话开始时，自动加载中期记忆和长期记忆
- 每个任务完成后，更新对应层级的记忆内容

---

## 第一层：短期记忆（会话级）

### 定义
当前会话内的即时状态，任务结束后清除（除非用户要求保留）

### 存储内容
- 当前创作章节编号
- 当前待处理任务清单
- 最近一次用户反馈/修改意见
- 会话中发现的伏笔或设定要点

### 更新时机
- 每次完成一个创作子任务后
- 用户提出新需求时（覆盖旧内容）
- 会话结束时可选择是否升级到中期记忆

### 示例格式
```
【短期记忆 · 当前会话】
- 任务：创作第5章
- 进度：已完成章节信息确认，故事梗概待用户选择
- 用户反馈：希望第5章增加"路途艰辛"的描写
- 新发现伏笔：赵虎对沈雾身份有疑虑（待后续利用）
```

---

## 第二层：中期记忆（项目级）

### 定义
当前小说项目的全局状态，跨会话保持

### 存储内容
- **项目基本信息**：书名、类型、当前章节进度
- **人物关系图谱**：核心人物+关系变化
- **当前主线矛盾**：进行中的核心冲突
- **伏笔追踪状态**：已埋设/已回收/待回收
- **设定一致性备忘**：容易出错的关键设定

### 更新时机
- 完成新章节后
- 用户确认新的剧情走向后
- 项目设定有大调整时
- 每月定期回顾更新

### 示例格式
```
【中期记忆 · 项目级 · 最后更新：2026-04-15】

项目：《和离后，我听见了废太子心声》
当前进度：第4章已完成，第5章待创作

核心人物状态：
- 沈雾：和离获得心声能力，跟随萧渊流放
- 萧渊：表面冷漠，暗中保护
- 青黛：随行丫鬟，发现有人监视
- 赵虎：押送官，暂时允许沈雾随行

主线进度：
- 阶段一（1-4章）：和离+心声觉醒 ✅
- 阶段二（5-20章）：流放之路 📋 进行中

伏笔状态（部分）：
- V001 同心蛊副作用：第1章埋设，第30章回收，进行中
- V003 萧承眼线：第2章暗示，第21章回收，待揭示
- V010 影卫组织：第3章出场，第8章详细展开

待处理：
- 第5章待创作
- 萧承眼线伏笔需在第5-20章中逐步加深暗示
```

## 第三层：长期记忆（用户偏好级）

### 定义
用户创作风格和个人偏好的持久记录，几乎不变化

### 存储内容
- **频道与题材偏好**：女频/古代言情/甜宠+虐心
- **核心技法偏好**：心声描写、推拉美学、五感描写
- **去AI味标准**：句式多样性、词汇离散度、节奏波动
- **写作雷区**：不喜欢的方式（待收集）
- **常用比喻库**：个人化的表达习惯（待建立）

### 更新时机
- 用户首次说明偏好时
- 用户对生成结果提出风格调整时
- 积累3次以上相同偏好反馈时更新

### 示例格式
```
【长期记忆 · 用户偏好级】

频道类型：女频 · 古代言情
风格倾向：甜虐交织（甜:虐 ≈ 6:4）
核心技法：
✓ 心声与言语反差（每章至少3处）
✓ 五感描写（尤其嗅觉/触觉）
✓ 推拉美学（感情线）
✓ 情感具象化（避免直接命名情感）

去AI味标准：
✓ 句式多样性（禁止连续3句相同结构）
✓ 词汇离散度（800字内3种表达）
✓ 节奏波动（长→短→超短交替）
✓ 语义毛刺（每千字0-2处）
✓ 微瑕疵注入（每2000字3处）

禁用清单：
✗ 直接命名情感（"她很悲伤"）
✗ 陈词滥调（"心如刀绞""月光如水"）
✗ AI高频词（"值得注意的是""此外"）

写作习惯：
- 章末钩子：悬念型/燃句型/反转型
- 字数目标：2500-3500字/章
- 叙事视角：第三人称限制（沈雾视角）
```

## 记忆调用流程

```
新会话开始
    │
    ▼
加载长期记忆（用户偏好）
    │
    ▼
加载中期记忆（项目状态）
    │
    ▼
加载短期记忆（如有未完成任务）
    │
    ▼
开始对话/任务
    │
    ├── 任务进行中
    │       │
    │       ▼
    │   定期更新短期记忆
    │       │
    │       ▼
    │   任务完成
    │
    ▼
任务/会话结束
    │
    ├── 判断是否需要升级到中期记忆
    │
    └── 更新对应层级
```

---

## 违规处理

如果跳过 Memory 工具的执行，将收到系统警告：
> ⚠️ Memory 工具未被执行。请使用 mcp_Memory 工具记录当前状态。
