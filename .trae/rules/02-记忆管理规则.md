# 记忆管理规则（结构化版）

## 核心原则

**用结构化格式记录，用标签化管理，用自动化更新。**

---

## 三层记忆架构

### 第一层：短期记忆（会话级）

**存储位置**：每次对话开始时加载，结束时清除

**内容格式**：
```yaml
短期记忆:
  当前任务: "写第5章"
  任务进度: "已完成场景设计，待写正文"
  用户反馈: "希望增加萧渊的脆弱面"
  新发现: 
    - "青黛发现影七监视"
    - "萧渊内功只恢复五成"
  待确认: "第5章是否加入副作用描写"
```

**更新时机**：
- 每完成一个子任务
- 用户提出新需求
- 发现新的伏笔/设定

---

### 第三层：长期记忆（用户偏好级）

**存储位置**：Memory工具 - 用户实体

**内容格式**：
```yaml
用户偏好:
  频道类型: "女频"
  题材: "古代言情"
  风格: "甜虐交织(甜:虐=6:4)"
  文风偏好: "标准"  # 可选值: "标准" / "Priest风格" / "其他"
  
  核心技法:
    - "心声与言语反差"
    - "五感描写(尤其嗅觉/触觉)"
    - "推拉美学"
    - "情感具象化"
    
  去AI味标准:
    句式: "禁止连续3句相同结构"
    词汇: "800字内同概念3种表达"
    节奏: "长→短→超短交替"
    毛刺: "每千字0-2处语义毛刺"
    瑕疵: "每2000字3处微瑕疵"
    
  禁用清单:
    - "直接命名情感(她很悲伤)"
    - "陈词滥调(心如刀绞)"
    - "AI高频词(值得注意的是)"
    
  写作习惯:
    字数: "2500-3500字/章"
    视角: "第三人称限制(沈雾视角)"
    钩子: "章末必留钩子"
    
  常用标签:
    - "💕 情感爽点"
    - "⚔️ 动作/冲突"
    - "🧠 智斗"
    - "💔 虐点"
    - "⏳ 待开始"
    - "📋 进行中"
    - "✅ 已完成"
```

**更新时机**：
- 用户首次说明偏好
- 用户反馈风格调整（3次以上相同反馈）
- 每月回顾

## 记忆调用流程

```
新会话开始
  │
  ▼
加载长期记忆 → 确定用户偏好
  │
  ▼
加载中期记忆 → 确定项目状态
  │
  ▼
加载短期记忆 → 确定当前任务
  │
  ▼
开始对话
  │
  ├── 任务进行中
  │       │
  │       ▼
  │   实时更新短期记忆
  │       │
  │       ▼
  │   任务完成
  │
  ▼
判断升级:
  - 短期 → 中期（章节完成/重大发现）
  - 中期 → 长期（风格确认/偏好稳定）
```

## 快速查询命令

| 查询需求 | 操作 |
|---------|------|
| "上一章写了什么" | 查中期记忆-主线进度 |
| "伏笔V001回收了吗" | 查中期记忆-伏笔追踪 |
| "萧渊现在什么状态" | 查中期记忆-人物状态 |
| "下一章该写什么爽点" | 查中期记忆-爽点规划 |
| "我喜欢什么风格" | 查长期记忆-用户偏好 |

## 记忆冲突检测

**自动检测场景**：
1. 新人物设定与已有设定冲突 → 警告并询问
2. 伏笔回收时间不合理（埋设第1章，回收第2章）→ 警告
3. 人物OOC（做出不符合性格的行为）→ 警告
4. 世界观设定前后矛盾 → 警告

**处理方式**：
- 冲突时列出矛盾点
- 给出修改建议
- 等待用户确认
