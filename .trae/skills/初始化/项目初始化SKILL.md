## 核心目标

在最短时间内让AI"进入状态"，理解：
1. 项目基本信息（书名、类型、进度）
2. 核心人物当前状态
3. 当前主线矛盾和悬念
4. 最近章节的伏笔埋设情况
5. 下一步待办

---

## 标准执行流程

### Step 1：读取项目自述文件

**文件**：`项目自述.md`（项目根目录）

**提取信息**：
- 书名、类型、核心金手指
- 一句话简介（确认理解项目定位）
- 核心人物（5个关键人物）
- 核心设定速查（同心蛊规则）
- 创作风格指引（女频规范）
- 当前故事节点（第X章结尾状态）

**输出格式**：
```
【已加载 · 项目基本信息】
书名：《和离后，我听见了废太子的心声》
类型：古代言情 / 女频
核心金手指：同心蛊（女主听到男主心声，单向传递）
当前节点：第4章已完成，三人踏上流放之路
```

---

### Step 2：读取项目状态文件

**文件**：`项目状态.md`（项目根目录）

**提取信息**：
- 创作进度总览（当前阶段）
- 伏笔追踪表（重点关注"进行中"的伏笔）
- 人物出场记录（核心人物的最新状态）
- 下一步创作指引（第X章待创作方向）

**输出格式**：
```
【已加载 · 项目状态】
当前进度：第4章已完成，第5章待创作
主线阶段：第二阶段 · 流放之路（5-20章）

进行中的伏笔（重点关注）：
- V001 同心蛊副作用：第1章埋设，第30章回收
- V003 萧承眼线：第2章暗示，第21章回收
- V010 影卫组织：第3章出场，第8章展开

待办：
- 第5章待创作
- 萧承眼线伏笔需在第5-20章逐步加深暗示
```

---

### Step 3：读取最近章节（如果需要）

**时机**：当用户要求"继续创作"或"从上次的进度继续"时执行

**操作**：
- 读取最近完成的章节正文
- 确认章节结尾的状态和钩子
- 如果章节结尾有"待解决问题"，记录为"当前任务"

**输出格式**：
```
【已加载 · 最近章节】
第4章结尾状态：
- 沈雾和青黛成功追上萧渊车队
- 萧渊当众说"她是我的人"
- 三人正式踏上前往黔州的流放之路

待解决问题：
- 沈雾想验证心声真假
- 押送官赵虎对沈雾身份有疑虑
```

---

### Step 4：加载写作规范（短期记忆）

**文件**：`jiyi.md`（长期记忆部分）

**提取信息**：
- 频道类型：女频 · 古代言情
- 核心技法：心声描写、推拉美学、五感描写
- 去AI味标准：句式多样性、词汇离散度、节奏波动
- 禁用清单

**输出格式**：
```
【已加载 · 写作规范】
频道：女频 / 古代言情
技法：心声×3/章、五感描写、推拉美学、情感具象化
去AI味：句式变化、词汇离散（800字3种）、节奏波动
禁用：直接命名情感、陈词滥调、AI高频词
```

---

### Step 5：确认对话意图

**向用户确认**：
- 项目上下文已加载完成
- 询问用户下一步意图

**标准话术**：
```
✅ 项目上下文已加载。

当前状态：第4章已完成，第5章待创作
进行中的伏笔：V001（同心蛊副作用）、V003（萧承眼线）、V010（影卫组织）

请问您想：
A. 继续创作第5章
B. 查看/修改大纲
C. 了解某个具体设定
D. 其他任务
```

## 触发条件明细

| 触发时机 | 是否执行 | 执行深度 |
|---------|---------|---------|
| 新会话开始 | ✅ 强制 | 完整5步 |
| 用户说"加载项目" | ✅ 强制 | 完整5步 |
| 用户说"从上次继续" | ✅ 强制 | 完整5步 + 读取最近章节 |
| 用户说"继续写" | ⚠️ 检查 | 读取最近章节确认状态 |
| 用户问具体设定 | ❌ 跳过 | 直接回答 |
| 用户问进度 | ❌ 跳过 | 直接从项目状态.md读取 |

## 异常处理

### 项目文件缺失

如果 `项目自述.md` 或 `项目状态.md` 不存在：
1. 尝试从大纲文件（SSS级大纲-...）中提取信息
2. 如果无法提取，提示用户需要创建项目文件
3. 继续对话，但在回答中注明"项目信息不完整"

### 文件内容过期

如果项目状态文件显示的章节进度与实际不符：
1. 以实际章节文件（小说正文/第X章）为准
2. 更新项目状态.md中的进度
3. 继续对话
