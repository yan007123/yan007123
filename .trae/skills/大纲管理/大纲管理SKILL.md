## 核心目标

1. **伏笔追踪**：管理所有伏笔的状态（已埋设、已回收、进行中）
2. **章节管理**：追踪每个章节的完成状态和内容摘要
3. **大纲更新**：在大纲变更时保持一致性
4. **冲突预警**：发现伏笔矛盾或大纲漏洞

---

## 四大功能模块

---

## 模块一：伏笔追踪

### 查看伏笔状态

**命令**：`查看伏笔` / `伏笔状态`

**操作**：
1. 读取 `项目状态.md` 中的伏笔追踪表
2. 按状态分类展示
3. 标注即将到期的伏笔（计划在最近3章回收的）

**输出格式**：
```
【伏笔追踪表】

🔴 紧急（最近3章需回收）：
- V003 萧承眼线：计划第21章，距当前还有16章

🟡 进行中（按计划埋设中）：
- V001 同心蛊副作用：第1章埋设，第30章回收
- V010 影卫组织：第3章出场，第8章展开

🟢 已回收：
（暂无）

⚪ 待埋设（尚未开始）：
- V012 四皇子真实身份：计划40章回收
```

### 埋设伏笔

**命令**：`埋设伏笔 [描述]` / `添加伏笔 [描述]`

**操作**：
1. 确认当前章节编号
2. 询问伏笔内容、计划回收章节
3. 更新 `项目状态.md` 中的伏笔追踪表
4. 确认埋设位置和方式

**交互流程**：
```
用户：埋设一个新伏笔，关于"沈雾的苗疆血统"
AI：
1. 这是V002（第1章已暗示），还是新增？
2. 计划在哪章回收？
3. 以什么形式回收？（揭示/事件触发/角色揭露）

用户：这是V002，已经埋过了，在第25章回收
AI：
✅ 已确认
更新：V002 状态：第1章（暗示）→ 进行中
计划回收：第25章
```

### 回收伏笔

**命令**：`回收伏笔 [ID]` / `[伏笔ID]已回收`

**操作**：
1. 确认伏笔ID
2. 确认回收的章节编号
3. 更新伏笔状态为"已回收"
4. 可选：记录回收方式摘要

**输出格式**：
```
✅ 【伏笔已回收】
ID：V006
伏笔：柳氏是萧承棋子
回收章节：第10章
方式：在萧承信件中揭露

伏笔追踪表已更新：
V006：🔴 紧急 → 🟢 已回收
```

## 模块二：章节管理

### 查看章节状态

**命令**：`查看章节` / `章节状态` / `创作进度`

**操作**：
1. 读取 `项目状态.md` 中的创作进度总览
2. 读取 `小说正文/` 目录获取实际文件列表
3. 交叉比对，标注不一致

**输出格式**：
```
【创作进度】

✅ 已完成：
- 第1章 和离
- 第2章 无处可归
- 第3章 决心追随
- 第4章 雪地重逢

📋 进行中：
- 第5章（待创作）

📋 待创作：
- 第6-20章（流放之路阶段）
- 第21-30章（扎根黔州阶段）

⚠️ 状态不一致：
- 项目状态.md显示"第4章已完成"
- 但小说正文/目录中未找到第4章文件
```

### 更新章节状态

**命令**：`第X章已完成` / `更新章节状态`

**操作**：
1. 确认章节编号
2. 询问章节名称和核心事件摘要
3. 更新 `项目状态.md` 中的进度
4. 可选：更新人物出场记录

**交互流程**：
```
用户：第4章已完成
AI：
请提供：
1. 章节名称：____________
2. 核心事件（1-2句）：____________
3. 本章埋设的伏笔（如有）：____________

用户：雪地重逢；沈雾追上萧渊请求随行流放
AI：
✅ 第4章状态已更新：
- 章节名：雪地重逢
- 核心事件：沈雾追上萧渊，请求随行流放
- 下一章：第5章待创作
```

## 模块三：大纲更新

### 查看大纲

**命令**：`查看大纲` / `看大纲` / `大纲是什么`

**操作**：
1. 读取 `项目自述.md` 中的大纲文件索引
2. 读取 `SSS级大纲-....md` 中的章节规划
3. 按阶段展示大纲概要

**输出格式**：
```
【大纲概要】

第一阶段：和离与心声觉醒（1-4章）✅
- 核心：沈雾和离，获得同心蛊能力

第二阶段：流放之路（5-20章）📋
- 核心：路途艰辛，感情推进，眼线揭露
- 关键节点：
  * 第10章：柳氏身份完整揭露
  * 第15章：沈家与萧承勾连揭露
  * 第18章：谢家联络人出现

第三阶段：扎根黔州（21-30章）📋
- 核心：荒院生活，反击开始，副作用显现
- 关键节点：
  * 第21章：驿站老板眼线身份揭露
  * 第25章：玉坠秘密揭露
  * 第30章：副作用真相揭露
```

### 调整大纲

**命令**：`调整大纲` / `修改大纲` / `大纲改动`

**操作**：
1. 确认要调整的位置（章节编号）
2. 确认调整内容（前/后期影响）
3. 更新对应的大纲文件
4. 更新 `项目状态.md` 中的伏笔追踪（如果涉及）

**警告模板**：
```
⚠️ 【大纲调整警告】
调整位置：第15章（原计划沈家揭露）
影响范围：
- 可能影响：V007（沈家与萧承勾连）
- 可能影响：第20章谢家出场时机

确认调整吗？
```

## 模块四：冲突预警

### 自动检查

**触发时机**：
- 完成新章节后
- 大纲调整后
- 用户要求"检查大纲"时

**检查项目**：
| 检查项 | 说明 |
|-------|------|
| 伏笔冲突 | 两个伏笔要求在同一事件中出现矛盾 |
| 时间线冲突 | 两个事件的时间顺序矛盾 |
| 人物冲突 | 同一人出现在两个不同地点同时 |
| 能力冲突 | 金手指能力描述前后不一 |
| 回收冲突 | 伏笔回收方式与前期埋设矛盾 |

**发现问题时的输出格式**：
```
🔴 【冲突预警】
冲突类型：时间线冲突
涉及事件：
- 第8章计划：影七首次正式亮相
- 第7章计划：影七暗中保护沈雾

问题：第7章影七出现后，第8章才是正式亮相，时间太近

建议：
A. 第7章改为"暗示影七存在"（不正式亮相）
B. 第8章改为"影七正式登场"
C. 调整其中一个章节的内容

请确认如何处理。
```

## 触发条件明细

| 触发时机 | 是否执行 | 执行模块 |

| 用户要求"查看大纲" | ✅ | 模块一/二 |
| 用户要求"追踪伏笔" | ✅ | 模块一 |
| 用户要求"更新进度" | ✅ | 模块二 |
| 用户要求"调整大纲" | ✅ | 模块三 |

## 文件依赖

- `项目状态.md`（伏笔追踪表、章节状态）
- `项目自述.md`（大纲文件索引）
- `SSS级大纲-...-完整修订版.md`（主大纲）
- `SSS级大纲-...-1-20章细纲-Part1.md`（细纲1）
- `SSS级大纲-...-21-30章细纲-Part2.md`（细纲2）
