# 大纲管理专业指南

## 核心理念

**大纲是地图，不是牢笼。**

好的大纲让你不迷路，但也允许你在路上发现新的风景。

---

## 第一章：大纲三层结构

### 第一层：故事核（1页）

**内容**：一句话概括你的故事

**公式**：
```
【主角】是一个【身份】的人，
TA想要【目标】，
但【阻碍】挡住了TA，
于是TA【行动】，
最终【结果】。
```

**示例**：
```
林晚是一个被家族抛弃的千金，
她想要夺回属于自己的一切，
但继母和妹妹处处陷害她，
于是她联手神秘总裁步步为营，
最终不仅复仇成功，还收获了真爱。
```

**作用**：
- 明确故事核心
-  pitch给编辑或读者
- 检查故事是否完整

---

### 第二层：主线大纲（5-10页）

**内容**：故事的主要脉络

**结构**：

```
【开端】（0-20%）
- 世界观介绍
- 主角登场
- 触发事件
- 主要人物出场

【发展】（20-60%）
- 主角行动
- 遭遇阻碍
- 结识盟友
- 感情发展
- 小高潮

【高潮】（60-90%）
- 最大危机
- 最终对决
- 真相揭露
- 情感爆发

【结局】（90-100%）
- 问题解决
- 人物归宿
- 主题升华
```

**每个部分包含**：
- 主要事件（发生了什么）
- 人物变化（主角的成长）
- 感情进展（CP的关系变化）
- 悬念设置（留下的钩子）

---

### 第三层：细纲/章纲（每章1段）

**内容**：每章的具体内容

**格式**：
```
第X章：章节标题
【场景】地点、时间
【人物】出场角色
【目标】这章要完成什么
【内容】具体情节
【钩子】章末悬念
```

**示例**：
```
第10章：雨夜的相遇
【场景】医院门口，雨夜
【人物】女主、男主
【目标】让男女主正式认识，建立第一印象
【内容】女主加班到深夜，下雨没带伞，
      在公交站躲雨。男主的车停在面前，
      让她上车。女主犹豫后上车，
      发现男主是白天面试她的总裁。
【钩子】男主说："我对你很感兴趣。"
```

## 第二章：大纲撰写流程

### 步骤1：确定故事核（30分钟）

**问题清单**：
1. 主角是谁？有什么特别之处？
2. 主角想要什么？（目标）
3. 什么在阻止TA？（阻碍）
4. 谁会帮助/阻碍TA？（人物）
5. 故事的终点是什么？（结局）

**输出**：故事核一句话

---

### 步骤2：搭建主线框架（2-3小时）

**方法**：
1. 确定故事类型（甜宠/虐恋/复仇/成长）
2. 列出必须发生的大事件（10-15个）
3. 按时间顺序排列
4. 分配到大纲四部分

**工具**：便利贴法
- 每个事件写在便利贴上
- 在桌面/白板上排列
- 调整顺序直到流畅

---

### 步骤3：设计人物（2-3小时）

**主角设计**：
- 起点状态
- 终点状态
- 成长路径

**配角设计**：
- CP：和主角的关系发展
- 反派：动机和行动
- 助攻：功能和出场时机

---

### 步骤4：撰写细纲（持续进行）

**建议**：
- 先写前30章细纲
- 边写边补充后续
- 每完成10章，更新一次大纲

**细纲粒度**：
- 开头：详细（每章都要细纲）
- 中间：中等（每3-5章一个概括）
- 结尾：粗略（知道大致方向即可）

## 第三章：大纲管理技巧

### 文件结构建议

```
项目文件夹/
├── 大纲/
│   ├── 故事核.md          # 一句话概括
│   ├── 主线大纲.md        # 四部分结构
│   ├── 细纲.md           # 章纲
│   └── 人物关系图.md      # 关系网
├── 设定/
│   ├── 世界设定.md
│   ├── 人物设定.md
│   └── 时间线.md
├── 章节/
│   ├── 第1章.md
│   ├── 第2章.md
│   └── ...
└── 备忘/
    ├── 灵感记录.md
    ├── 修改记录.md
    └── 问题清单.md
```

### 大纲更新原则

#### 什么时候更新大纲？

1. **写作前**：根据细纲写正文
2. **写作中**：发现更好的想法时
3. **每10章**：回顾并调整后续
4. **卡文时**：检查大纲是否合理
5. **大事件后**：确保后续连贯

#### 如何修改大纲？

**原则**：
- 小修改随时进行
- 大修改要慎重（影响后续）
- 记录修改原因
- 保留旧版本（对比）

**修改步骤**：
1. 在副本上修改
2. 检查连锁反应
3. 更新相关章节
4. 记录修改日志

---

### 伏笔管理

#### 伏笔记录表

| 伏笔 | 埋设章节 | 揭晓章节 | 状态 |
|------|----------|----------|------|
| 女主身世 | 第5章 | 第50章 | 已埋 |
| 男主旧伤 | 第12章 | 第45章 | 已埋 |
| 神秘人身份 | 第20章 | 第60章 | 待揭晓 |

**原则**：
- 埋设时就要想好揭晓时机
- 不要埋太多（读者记不住）
- 揭晓要有冲击力
- 遗忘的伏笔要及时回收或删除

## 第四章：常见问题解决

### 问题1：写不下去（卡文）

**诊断**：
- [ ] 细纲不够详细？
- [ ] 人物动机不明确？
- [ ] 情节逻辑有问题？
- [ ] 节奏太紧/太松？

**解决**：
1. 回到大纲，检查这章的目标
2. 和人物对话：TA现在最想要什么？
3. 跳过这章，写后面有灵感的部分
4. 增加一个小事件过渡

---

### 问题2：偏离大纲

**判断**：
- 偏离是更好的发展 → 更新大纲
- 偏离是跑题 → 拉回来
- 偏离是偷懒 → 坚持原大纲

**方法**：
1. 暂停写作
2. 对比原大纲和新想法
3. 评估哪个更好
4. 做出决定并执行

---

### 问题3：大纲太死板

**症状**：
- 严格按照大纲写，没有惊喜
- 人物像提线木偶
- 情节机械推进

**解决**：
1. 大纲只定方向，不定细节
2. 允许人物"自己决定"行动
3. 保留20%的即兴空间
4. 定期问自己：这是人物会做的吗？

---

### 问题4：没有大纲不敢写

**症状**：
- 过度规划，迟迟不动笔
- 害怕写得不够好
- 完美主义 paralysis

**解决**：
1. 先写故事核和开头10章细纲
2. 开始写，边写边完善
3. 接受第一稿不完美
4. 完成比完美重要
