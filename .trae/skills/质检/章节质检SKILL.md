# 章节质检 SKILL

> 该SKILL在章节内容创作完成后自动触发（或用户要求"检查章节质量"时），用于发现逻辑漏洞、人名错误、前后矛盾等问题。
> **触发条件**：章节创作完成后、用户要求"质检"、用户要求"检查逻辑"时执行。

---

## 核心目标

在章节内容输出前，发现并修正：
1. **逻辑漏洞**：物理不合理、因果矛盾
2. **一致性错误**：人名/年龄/性格/关系前后不一致
3. **时间线矛盾**：日期/季节/时辰与前文不符
4. **伏笔缺失**：该埋的伏笔没埋，或伏笔回收时机不对
5. **设定冲突**：与世界观/人物设定矛盾
6. **金手指滥用**：主角能力过强/不合理
7. **AI痕迹残留**：明显的AI写作特征

---

## 质检流程（5步）

### Step 1：准备质检材料

**输入**：待检查的章节正文

**操作**：
1. 提取章节中的所有人物名称
2. 提取章节中的时间/地点标记
3. 提取章节中涉及的关键事件
4. 标注章节在项目中的位置（章节编号）

**输出格式**：
```
【待质检章节】
章节：第4章 雪地重逢
字数：约2800字
时间：乾元十四年，腊月十一
地点：永定门外官道 → 流放路

【人物清单】
沈雾、萧渊、青黛、赵虎

【关键事件】
1. 沈雾追到永定门
2. 请求跟随流放
3. 萧渊当众说"她是我的人"
4. 赵虎允许随行

【涉及伏笔】
V001 同心蛊、V003 萧承眼线（暗示）、V010 影卫
```

---

### Step 2：人物一致性检查

**检查项目**：
| 检查项 | 说明 | 严重程度 |
|-------|------|---------|
| 人名错误 | 名字写错/写别字 | 🔴 致命 |
| 人物性格 | 与设定不符 | 🔴 致命 |
| 人物关系 | 关系描述与前文矛盾 | 🔴 致命 |
| 人物年龄 | 与设定不符 | 🟡 中等 |
| 人物外貌 | 与设定不符 | 🟡 中等 |
| 人物习惯 | 与设定矛盾 | 🟡 中等 |

**执行方法**：
1. 对照项目自述.md中的人物设定
2. 对照大纲中的人物发展弧线
3. 对照前文已建立的人物形象

**发现问题时的输出格式**：
```
⚠️ 【一致性错误】
位置：第3段第2句
问题：沈雾"笑着说话"
原因：人设中沈雾性格清冷，在悲伤/紧张时不会笑
建议：改为"扯了扯嘴角"或"语气平淡"
```

---

### Step 3：时间线与逻辑检查

**检查项目**：
| 检查项 | 说明 | 严重程度 |
|-------|------|---------|
| 时间矛盾 | 早上发生的事写成午后的阳光 | 🔴 致命 |
| 物理不合理 | 一步跨过三丈宽的沟 | 🟡 中等 |
| 因果缺失 | 没有铺垫突然出现结果 | 🟡 中等 |
| 能力越界 | 主角突然具备不可能的能力 | 🔴 致命 |
| 数字错误 | 与前文数字矛盾 | 🟡 中等 |

**执行方法**：
1. 梳理本章的时间线（按场景/事件顺序）
2. 检查物理场景的合理性
3. 追踪本章事件的前因后果

**发现问题时的输出格式**：
```
⚠️ 【时间线问题】
位置：第5段
问题："午后的阳光"——前文写的是"辰时出发"，没有交代通勤过程
建议：改为"上午的日光"或补充通勤过程的描写
```

---

### Step 4：伏笔与钩子检查

**检查项目**：
| 检查项 | 说明 | 严重程度 |
|-------|------|---------|
| 伏笔缺失 | 计划埋设的伏笔没有出现 | 🔴 致命 |
| 伏笔提前 | 伏笔不该这么早揭示 | 🟡 中等 |
| 钩子缺失 | 章末没有钩子 | 🔴 致命 |
| 钩子质量 | 钩子太弱，引力不足 | 🟡 中等 |
| 新伏笔无意识埋设 | 埋了新伏笔但没有记录 | ⚪ 提示 |

**执行方法**：
1. 对照项目状态.md中的伏笔追踪表
2. 确认本章应该埋设/回收哪些伏笔
3. 检查章末钩子的类型和质量

**发现问题时的输出格式**：
```
⚠️ 【伏笔问题】
计划：本章应加深V003（萧承眼线）的暗示
状态：本章未出现任何眼线相关暗示
建议：在路途中加入"有人在暗中观察"的细节
```

---

### Step 5：设定与世界观检查

**检查项目**：
| 检查项 | 说明 | 严重程度 |
|-------|------|---------|
| 世界观矛盾 | 地理/制度与设定不符 | 🔴 致命 |
| 金手指违规 | 同心蛊能力描述错误 | 🔴 致命 |
| 等级/境界错误 | 修仙类设定等级矛盾 | 🔴 致命 |
| 物品/法宝描述错误 | 与设定不符 | 🟡 中等 |
| 势力分布错误 | 与设定矛盾 | 🟡 中等 |

**执行方法**：
1. 对照项目自述.md中的同心蛊规则
2. 对照大纲中的世界观设定
3. 检查本章出现的物品/能力是否符合设定

**发现问题时的输出格式**：
```
⚠️ 【设定错误】
位置：第7段
问题：沈雾在"千米之外"听到了萧渊的心声
原因：同心蛊设定中，超过1000米心声无效
建议：缩短距离，或改为"隐约感知到情绪波动"
```

## 质检报告模板

### 无问题报告
```
✅ 【质检通过】
章节：第X章
问题数：0

通过检查：
✓ 人物一致性
✓ 时间线逻辑
✓ 伏笔埋设
✓ 世界观设定
✓ 章末钩子

可以输出。
```

### 有问题报告
```
⚠️ 【质检发现问题】
章节：第X章
问题数：3

🔴 致命问题（必须修正）：
1. 人名错误：第3段"萧渊"写成"萧远"
2. 钩子缺失：本章结尾没有钩子

🟡 中等问题（建议修正）：
3. 时间线：第5段"午后的阳光"与前文矛盾

请修正后再输出。
```

## 与人味SKILL的分工

| 检查类型 | 质检SKILL | 人味SKILL |
|---------|----------|----------|
| 人物一致性 | ✅ 负责 | - |
| 时间线逻辑 | ✅ 负责 | - |
| 伏笔检查 | ✅ 负责 | - |
| 设定检查 | ✅ 负责 | - |
| 句式多样性 | - | ✅ 负责 |
| 词汇离散度 | - | ✅ 负责 |
| 节奏波动 | - | ✅ 负责 |
| 情感描写 | - | ✅ 负责 |
| AI痕迹 | - | ✅ 负责 |

**执行顺序**：质检SKILL（逻辑/一致性）→ 人味SKILL（风格/去AI味）

## 异常处理

### 无法确定问题时

如果某处"看起来不对但无法确定"：
1. 标记为"疑似问题"
2. 说明为什么觉得可疑
3. 让用户判断是否需要修正

### 与大纲冲突时

如果章节内容与大纲有偏差：
1. 检查是否是大纲本身需要调整
2. 如果是章节写错了，指出具体问题
3. 如果是大纲需要更新，提示用户确认

### 发现重大问题时

如果发现🔴致命问题：
1. 立即停止输出
2. 输出详细的问题报告
3. 等待用户确认如何修正
