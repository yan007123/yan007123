# 智能细纲管理指南

## 核心理念

**细纲不是孤立的，它连接着过去和未来。**

好的细纲管理需要：
- 回顾：之前发生了什么？
- 理解：现在在哪里？
- 前瞻：未来要去哪里？
- 连接：如何把现在和未来连起来？

---

## 第一章：上下文感知框架

### 信息来源层级

当规划细纲时，需要读取和分析以下信息：

#### 第一层：项目结构（必须读取）

```
工作目录/
├── 大纲/
│   ├── 主线大纲.md      → 了解整体走向
│   ├── 细纲.md         → 查看已有规划
│   └── 人物关系图.md    → 理解人物关系
├── 设定/
│   ├── 世界设定.md      → 了解世界观
│   ├── 人物设定.md      → 查看人物档案
│   └── 时间线.md        → 掌握时间顺序
├── 章节/               → 已写章节内容
│   ├── 第一章.txt
│   ├── 第二章.txt
│   └── ...
└── 备忘录/             → 灵感和待办
    └── 备忘录.md
```

**读取优先级**：
1. 当前进度（写到哪章了）
2. 主线大纲（整体方向）
3. 已有细纲（后续规划）
4. 最近3-5章内容（上下文）
5. 人物设定（角色状态）

---

#### 第二层：上下文要素

**必须考虑的上下文信息**：

| 要素 | 说明 | 影响 |
|------|------|------|
| **当前进度** | 写到第几章了 | 确定细纲起始点 |
| **人物状态** | 角色当前的情绪、关系、目标 | 决定人物行动 |
| **未解决冲突** | 悬而未决的问题 | 需要推进或解决 |
| **已埋伏笔** | 之前设置的悬念 | 需要维护或回收 |
| **感情阶段** | CP处于哪个阶段 | 决定感情戏内容 |
| **世界观限制** | 设定中的规则 | 不能违背的约束 |

### 上下文分析流程

#### 步骤1：定位当前位置

**问题清单**：
1. 最新写完的是第几章？
2. 那章的结尾发生了什么？（钩子）
3. 主要人物现在的状态是？
4. 当前最大的悬念是？

**输出**：当前位置快照

```
【当前位置】第15章结束
【人物状态】
- 女主：刚知道真相，情绪崩溃
- 男主：追出去安慰，关系缓和
- 反派：正在策划下一步行动
【未解决冲突】
- 女主身世之谜（已埋，待揭晓）
- 公司危机（进行中）
- 男主旧伤复发（伏笔）
【感情阶段】暧昧期→即将突破
【下章钩子】男主说："我有话要对你说"
```

---

#### 步骤2：回顾主线目标

**问题清单**：
1. 按照大纲，第16-20章应该完成什么？
2. 这个阶段的感情线目标是什么？
3. 有什么大事件需要铺垫？
4. 伏笔什么时候揭晓？

**输出**：阶段目标

```
【阶段目标】第16-20章
【剧情目标】
- 女主振作，开始反击
- 公司危机升级
- 反派露出马脚
【感情目标】
- 暧昧期突破，关系确认
- 第一次正式约会
【铺垫目标】
- 为第25章的大危机埋伏笔
- 引出男二角色
```

---

#### 步骤3：连接现在与未来

**核心问题**：
从"第15章结束"到"第20章目标"，中间需要发生什么？

**方法**：倒推法 + 正推法

**倒推法**（从目标往回推）：
```
第20章目标：关系确认
    ↑ 需要：表白场景
    ↑ 需要：暧昧氛围达到顶点
    ↑ 需要：独处机会+情绪高潮
    ↑ 需要：共同经历某个事件
```

**正推法**（从当前往后推）：
```
第15章结束：男主说"我有话要说"
    ↓ 第16章：男主表白？不，先延迟满足
    ↓ 第17章：意外打断，暧昧升级
    ↓ 第18章：共同面对危机，感情升温
    ↓ 第19章：独处，氛围到位
    ↓ 第20章：确认关系
```

## 第二章：细纲智能生成

### 单章细纲模板

```markdown
### 第X章：[章节标题]

#### 基础信息
- **场景**：地点、时间、天气
- **人物**：出场角色及状态
- **字数预估**：3000-5000字

#### 本章目标
- **剧情目标**：这章要推动什么情节
- **感情目标**：CP关系有什么进展
- **人物目标**：角色有什么成长/变化

#### 承接上文
- 上一章钩子：[复述钩子]
- 需要回应：[哪些悬念要处理]
- 人物状态：[角色带着什么情绪入场]

#### 具体情节
1. [场景1]：发生了什么
   - 动作/对话
   - 情绪变化
   - 信息透露

2. [场景2]：
   ...

#### 章末钩子
- **类型**：悬念型/情绪型/冲突型/亲密型
- **内容**：具体钩子内容
- **作用**：为什么让读者想看下章

#### 伏笔与铺垫
- **埋设**：这章要埋什么伏笔
- **呼应**：呼应之前的哪个伏笔
- **限制**：不能违背的设定

#### 写作提示
- **节奏**：快/中/慢
- **重点**：这章最重点写什么
- **注意**：容易写崩的地方
```

### 批量细纲生成

#### 场景：需要规划接下来10章

**步骤1：分析上下文**
- 读取最近3章内容
- 查看主线大纲对应部分
- 确认人物当前状态

**步骤2：确定阶段目标**
- 这10章要完成什么大事件？
- 感情线要推进到哪个阶段？
- 需要埋哪些伏笔？

**步骤3：设计章节链**

```
第16章：延迟满足
- 承接：男主"有话要说"
- 内容：被打断，女主误会
- 钩子：发现神秘信件

第17章：误会升级
- 承接：信件内容引发误会
- 内容：女主躲避男主
- 钩子：看到男主和别的女人

第18章：真相大白
- 承接：误会达到顶点
- 内容：解释清楚，关系破冰
- 钩子：共同面对危机

...
```

**步骤4：检查连贯性**
- 每章都承接上一章钩子了吗？
- 人物行为符合人设吗？
- 节奏有起伏吗？
- 感情线在推进吗？

## 第三章：细纲管理功能

### 功能1：续写细纲

**场景**：已经写了20章，需要规划21-30章

**操作流程**：
1. 读取第20章内容（了解当前状态）
2. 查看主线大纲（了解阶段目标）
3. 分析人物状态（确定行动逻辑）
4. 生成21-30章细纲
5. 检查与已有内容的连贯性

---

### 功能2：调整细纲

**场景**：写到第15章时发现更好的想法，需要调整后续

**操作流程**：
1. 分析新想法的影响范围
2. 检查连锁反应：
   - 人物关系变化？
   - 伏笔需要调整？
   - 大事件时间要改？
3. 修改相关章节细纲
4. 记录修改日志

**修改日志模板**：
```
【修改日期】2026-03-16
【修改章节】第15章后
【修改原因】发现让女主提前知道真相更有冲击力
【影响范围】第15-25章
【具体调整】
- 第16章：原计划男主隐瞒 → 改为女主质问
- 第17章：原计划误会 → 改为对峙
- 第18-20章：加速感情突破
【新增伏笔】女主开始调查过去
【待检查】第25章大危机是否需要提前
```

---

### 功能3：填补空白

**场景**：主线大纲有，但细纲缺失某些章节

**操作流程**：
1. 确定空白章节的位置
2. 查看前后章节的细纲
3. 理解这章需要完成的过渡功能
4. 生成合适的细纲
5. 确保与前后连贯

---

### 功能4：优化细纲

**场景**：已有细纲，但需要提升质量

**检查清单**：
- [ ] 每章都有明确的钩子吗？
- [ ] 人物行为符合当前状态吗？
- [ ] 节奏有变化吗？（避免连续快节奏或慢节奏）
- [ ] 感情线在稳步推进吗？
- [ ] 伏笔得到维护了吗？
- [ ] 有冗余章节可以合并吗？
- [ ] 有过紧的章节需要拆分吗？

## 第四章：上下文关联技巧

### 人物状态追踪

**人物状态表**（需要维护）：

| 人物 | 当前情绪 | 目标 | 关系状态 | 待解决问题 |
|------|----------|------|----------|------------|
| 女主 | 迷茫→振作 | 查明真相 | 对男主有好感但犹豫 | 身世之谜 |
| 男主 | 心疼→坚定 | 保护女主 | 已确定心意 | 旧伤隐患 |
| 反派 | 得意→警惕 | 除掉女主 | 表面和善 | 阴谋败露风险 |

**使用**：每规划一章前，更新人物状态，确保行动符合逻辑。

---

### 伏笔状态追踪

**伏笔管理表**：

| 伏笔 | 埋设章节 | 当前状态 | 计划揭晓 | 相关章节 |
|------|----------|----------|----------|----------|
| 女主身世 | 第5章 | 已暗示 | 第50章 | 16,25,40 |
| 男主旧伤 | 第12章 | 偶尔提及 | 第45章 | 20,35 |
| 神秘人 | 第20章 | 刚出现 | 第60章 | 22,30,50 |

**使用**：规划细纲时检查，确保：
- 该维护的伏笔有提及
- 该揭晓的伏笔有准备
- 不遗忘重要伏笔

---

### 感情阶段追踪

**感情线进度表**：

```
阶段：暧昧期（第10-30章）
当前：第15章结束
进度：60%
状态：互相有好感，但未说破
下一步：需要推进到"即将确认"状态
关键事件：需要一个让关系质变的事件
```

**使用**：确保感情戏不拖沓也不冒进。

## 第五章：实战示例

### 示例：规划第21-25章细纲

**上下文分析**：

```
【当前位置】第20章结束
【上一章钩子】男主终于表白，女主没有回答
【人物状态】
- 女主：内心动摇，还没准备好
- 男主：忐忑等待答复
- 反派：得知两人关系，准备破坏
【阶段目标】
- 感情：确认关系
- 剧情：公司危机爆发
- 铺垫：为第30章大高潮准备
```

**生成细纲**：

```markdown
### 第21章：犹豫
**承接**：女主面对表白没有回答
**目标**：展示女主内心挣扎，延迟满足
**情节**：
1. 女主找借口离开，需要时间思考
2. 回忆过去，害怕受伤
3. 闺蜜开导，但女主仍犹豫
**钩子**：男主说："我等你，不管多久"

### 第22章：危机初现
**承接**：女主还没答复，公司出问题
**目标**：转移注意力，增加外部压力
**情节**：
1. 公司项目出问题，女主焦头烂额
2. 男主暗中帮忙，但不留名
3. 女主发现是男主帮忙
**钩子**：女主看到男主疲惫的样子，心软

### 第23章：靠近
**承接**：女主开始动摇
**目标**：感情升温，准备突破
**情节**：
1. 女主主动约男主吃饭（感谢）
2. 两人独处，气氛暧昧
3. 女主差点说出心里话，被打断
**钩子**：反派出现，制造误会

### 第24章：误会
**承接**：反派出现，女主误会
**目标**：最后考验，让女主认清心意
**情节**：
1. 女主看到男主和反派"亲密"（设计好的）
2. 女主伤心，决定放弃
3. 男主发现女主误会，追出去解释
**钩子**：女主说："我不想再受伤了"

### 第25章：确认
**承接**：误会达到顶点
**目标**：感情突破，关系确认
**情节**：
1. 男主不顾一切解释清楚
2. 女主终于放下心防
3. 女主主动吻男主（或表白）
4. 关系正式确认
**钩子**：公司危机全面爆发，需要共同面对
```

**检查**：
- ✅ 承接了第20章钩子
- ✅ 感情从犹豫→确认，有进展
- ✅ 每章都有钩子
- ✅ 节奏：慢→快→快→快→高潮
- ✅ 埋设了公司危机伏笔
- ✅ 人物行为符合状态

## 第六章：使用指南

### 何时使用此技能

1. **规划新章节**：需要写接下来的N章
2. **调整已有细纲**：发现更好的想法
3. **填补细纲空白**：某些章节只有概要
4. **检查细纲质量**：确保连贯性和吸引力
5. **卡文时**：通过细纲找到方向

### 需要提供的信息

为了最好地帮助您，请提供：

**必需**：
- 当前写到第几章了？
- 最新一章的内容或钩子

**有帮助**：
- 主线大纲的关键节点
- 人物当前状态
- 这阶段的目标（剧情+感情）
- 需要埋设或回收的伏笔

### 输出格式

我会提供：
1. **上下文分析**：当前状态总结
2. **阶段目标**：这阶段要完成什么
3. **细纲**：每章的详细规划
4. **检查清单**：确保连贯性
5. **写作建议**：注意事项和技巧

---

## 使用场景

当用户需要：
1. **规划接下来N章** → 提供上下文感知的多章细纲
2. **调整已有细纲** → 分析影响并给出修改方案
3. **填补细纲空白** → 生成符合上下文的过渡章节
4. **优化细纲质量** → 检查并提供改进建议
5. **卡文找方向** → 分析上下文，给出继续写作的方案
