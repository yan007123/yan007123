# 章节创作 SKILL

> 该SKILL应在每次编写小说章节时自动调用，贯穿于章节创作的整个流程，从前到后，确保章节内容符合小说设定。

---

## 核心流程（8步）

### Step 1：章节信息确认

- 首先明确当前需要编写的章节编号
- 若未提供章节名称，主动向用户提问获取
- 确认章节字数目标（默认 2500-3000 字）

### Step 2：故事脉络分析

- 分析小说大纲以理解整体故事发展脉络
- 深入分析细纲中的剧情走向，特别关注当前章节前后的章节明细
- 梳理最近十章的详细概述及之前所有章节的概述
- 参考项目状态.md获取伏笔追踪和人物出场记录

### Step 3：故事梗概生成与确认

- 根据梳理的信息提供 5 个本章故事梗概供选择
- 若用户已提供故事梗概，则对其进行完善并请求用户确认
- 梗概要涵盖：核心事件、情感走向、心声互动设计、章末钩子

### Step 4：写作风格分析与应用

- 根据之前章节内容总结用户的写作风格和用词习惯
- 确认频道类型（女频/男频）
- 确认风格倾向（甜宠/虐心/正剧）
- **详细规则见：人味SKILL.md**

### Step 5：人物与场景设定梳理

- 确定本章需要出现的人物和场景
- 从项目设定档（项目自述.md）中获取特定人物的设定信息、行为习惯和语言习惯
- 在编写过程中严格遵守上述设定
- 检查人物出场记录，确保一致性

### Step 6：章节内容编写

- 在用户确认所有前期信息后，开始编写章节内容
- **编写时必须调用人味SKILL.md的规则**
- 确保每章包含：
  - 心声与言语反差（女频至少 3 处）
  - 五感描写（尤其嗅觉/触觉）
  - 情感具象化（避免直接命名情感）
  - 节奏波动（长段→短段→超短段）
  - 章末钩子（悬念/燃句/反转）

### Step 7：内容验证

- 检查章节内容是否符合故事背景
- 验证人物设定、人物行为及语言习惯是否准确
- 确认内容符合故事大纲和细纲走向
- 验证伏笔埋设是否按计划进行
- 若发现不符合之处，进行修正并重新验证

### Step 8：AI痕迹处理

- **详细规则见：人味SKILL.md**
- 每次正文创作到最后一步时，必须自动执行人味SKILL
- 对校正后的章节内容进行"去AI味"处理
- 执行终检协议（详见人味SKILL.md 第六层）

## 章节创作检查清单

### 前置检查
- [ ] 章节编号和名称已确认
- [ ] 已读取项目自述.md获取项目上下文
- [ ] 已读取项目状态.md获取伏笔追踪
- [ ] 已读取相关大纲和细纲
- [ ] 故事梗概已获用户确认

### 创作中执行
- [ ] 遵循人味SKILL.md的所有规则
- [ ] 每章至少 3 处心声与言语反差
- [ ] 五感描写（尤其嗅觉/触觉）
- [ ] 情感具象化
- [ ] 节奏波动（长→短→超短）
- [ ] 章末钩子已设计

### 创作后验证
- [ ] 人物设定一致性检查
- [ ] 时间线一致性检查
- [ ] 伏笔埋设/回收检查
- [ ] 去AI味终检（执行人味SKILL终检协议）

## 触发条件

本SKILL在以下情况自动触发：
- 用户要求"写第X章"
- 用户要求"继续创作"
- 用户要求"规划第X章"
