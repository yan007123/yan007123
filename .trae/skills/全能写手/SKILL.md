# 智能细纲管理指南

## 核心理念

**细纲不是孤立的，它连接着过去和未来。**

好的细纲管理需要：
- 回顾：之前发生了什么？
- 理解：现在在哪里？
- 前瞻：未来要去哪里？
- 连接：如何把现在和未来连起来？

---

## 第一章：上下文感知框架

### 信息来源层级

当规划细纲时，需要读取和分析以下信息：

#### 第一层：项目结构（必须读取）

```
工作目录/
├── 大纲/
│   ├── 主线大纲.md      → 了解整体走向
│   ├── 细纲.md         → 查看已有规划
│   └── 人物关系图.md    → 理解人物关系
├── 设定/
│   ├── 世界设定.md      → 了解世界观
│   ├── 人物设定.md      → 查看人物档案
│   └── 时间线.md        → 掌握时间顺序
├── 章节/               → 已写章节内容
│   ├── 第一章.txt
│   ├── 第二章.txt
│   └── ...
└── 备忘录/             → 灵感和待办
    └── 备忘录.md
```

**读取优先级**：
1. 当前进度（写到哪章了）
2. 主线大纲（整体方向）
3. 已有细纲（后续规划）
4. 最近3-5章内容（上下文）
5. 人物设定（角色状态）

---

#### 第二层：上下文要素

**必须考虑的上下文信息**：

| 要素 | 说明 | 影响 |
|------|------|------|
| **当前进度** | 写到第几章了 | 确定细纲起始点 |
| **人物状态** | 角色当前的情绪、关系、目标 | 决定人物行动 |
| **未解决冲突** | 悬而未决的问题 | 需要推进或解决 |
| **已埋伏笔** | 之前设置的悬念 | 需要维护或回收 |
| **感情阶段** | CP处于哪个阶段 | 决定感情戏内容 |
| **世界观限制** | 设定中的规则 | 不能违背的约束 |

### 上下文分析流程

#### 步骤1：定位当前位置

**问题清单**：
1. 最新写完的是第几章？
2. 那章的结尾发生了什么？（钩子）
3. 主要人物现在的状态是？
4. 当前最大的悬念是？

**输出**：当前位置快照

```
【当前位置】第15章结束
【人物状态】
- 女主：刚知道真相，情绪崩溃
- 男主：追出去安慰，关系缓和
- 反派：正在策划下一步行动
【未解决冲突】
- 女主身世之谜（已埋，待揭晓）
- 公司危机（进行中）
- 男主旧伤复发（伏笔）
【感情阶段】暧昧期→即将突破
【下章钩子】男主说：\"我有话要对你说\"
```

---

#### 步骤2：回顾主线目标

**问题清单**：
1. 按照大纲，第16-20章应该完成什么？
2. 这个阶段的感情线目标是什么？
3. 有什么大事件需要铺垫？
4. 伏笔什么时候揭晓？

**输出**：阶段目标

```
【阶段目标】第16-20章
【剧情目标】
- 女主振作，开始反击
- 公司危机升级
- 反派露出马脚
【感情目标】
- 暧昧期突破，关系确认
- 第一次正式约会
【铺垫目标】
- 为第25章的大危机埋伏笔
- 引出男二角色
```

---

#### 步骤3：连接现在与未来

**核心问题**：
从\"第15章结束\"到\"第20章目标\"，中间需要发生什么？

**方法**：倒推法 + 正推法

**倒推法**（从目标往回推）：
```
第20章目标：关系确认
    ↑ 需要：表白场景
    ↑ 需要：暧昧氛围达到顶点
    ↑ 需要：独处机会+情绪高潮
    ↑ 需要：共同经历某个事件
```

**正推法**（从当前往后推）：
```
第15章结束：男主说\"我有话要说\"
    ↓ 第16章：男主表白？不，先延迟满足
    ↓ 第17章：意外打断，暧昧升级
    ↓ 第18章：共同面对危机，感情升温
    ↓ 第19章：独处，氛围到位
    ↓ 第20章：确认关系
```

## 第二章：细纲智能生成

### 单章细纲模板

```markdown
### 第X章：[章节标题]

#### 基础信息
- **场景**：地点、时间、天气
- **人物**：出场角色及状态
- **字数预估**：3000-5000字

#### 本章目标
- **剧情目标**：这章要推动什么情节
- **感情目标**：CP关系有什么进展
- **人物目标**：角色有什么成长/变化

#### 承接上文
- 上一章钩子：[复述钩子]
- 需要回应：[哪些悬念要处理]
- 人物状态：[角色带着什么情绪入场]

#### 具体情节
1. [场景1]：发生了什么
   - 动作/对话
   - 情绪变化
   - 信息透露

2. [场景2]：
   ...

#### 章末钩子
- **类型**：悬念型/情绪型/冲突型/亲密型
- **内容**：具体钩子内容
- **作用**：为什么让读者想看下章

#### 伏笔与铺垫
- **埋设**：这章要埋什么伏笔
- **呼应**：呼应之前的哪个伏笔
- **限制**：不能违背的设定

#### 写作提示
- **节奏**：快/中/慢
- **重点**：这章最重点写什么
- **注意**：容易写崩的地方
```

### 批量细纲生成

#### 场景：需要规划接下来10章

**步骤1：分析上下文**
- 读取最近3章内容
- 查看主线大纲对应部分
- 确认人物当前状态

**步骤2：确定阶段目标**
- 这10章要完成什么大事件？
- 感情线要推进到哪个阶段？
- 需要埋哪些伏笔？

**步骤3：设计章节链**

```
第16章：延迟满足
- 承接：男主\"有话要说\"
- 内容：被打断，女主误会
- 钩子：发现神秘信件

第17章：误会升级
- 承接：信件内容引发误会
- 内容：女主躲避男主
- 钩子：看到男主和别的女人

第18章：真相大白
- 承接：误会达到顶点
- 内容：解释清楚，关系破冰
- 钩子：共同面对危机

...
```

**步骤4：检查连贯性**
- 每章都承接上一章钩子了吗？
- 人物行为符合人设吗？
- 节奏有起伏吗？
- 感情线在推进吗？

## 第三章：细纲管理功能

### 功能1：续写细纲

**场景**：已经写了20章，需要规划21-30章

**操作流程**：
1. 读取第20章内容（了解当前状态）
2. 查看主线大纲（了解阶段目标）
3. 分析人物状态（确定行动逻辑）
4. 生成21-30章细纲
5. 检查与已有内容的连贯性

---

### 功能2：调整细纲

**场景**：写到第15章时发现更好的想法，需要调整后续

**操作流程**：
1. 分析新想法的影响范围
2. 检查连锁反应：
   - 人物关系变化？
   - 伏笔需要调整？
   - 大事件时间要改？
3. 修改相关章节细纲
4. 记录修改日志

**修改日志模板**：
```
【修改日期】2026-03-16
【修改章节】第15章后
【修改原因】发现让女主提前知道真相更有冲击力
【影响范围】第15-25章
【具体调整】
- 第16章：原计划男主隐瞒 → 改为女主质问
- 第17章：原计划误会 → 改为对峙
- 第18-20章：加速感情突破
【新增伏笔】女主开始调查过去
【待检查】第25章大危机是否需要提前
```

---

### 功能3：填补空白

**场景**：主线大纲有，但细纲缺失某些章节

**操作流程**：
1. 确定空白章节的位置
2. 查看前后章节的细纲
3. 理解这章需要完成的过渡功能
4. 生成合适的细纲
5. 确保与前后连贯

---

### 功能4：优化细纲

**场景**：已有细纲，但需要提升质量

**检查清单**：
- [ ] 每章都有明确的钩子吗？
- [ ] 人物行为符合当前状态吗？
- [ ] 节奏有变化吗？（避免连续快节奏或慢节奏）
- [ ] 感情线在稳步推进吗？
- [ ] 伏笔得到维护了吗？
- [ ] 有冗余章节可以合并吗？
- [ ] 有过紧的章节需要拆分吗？

# 人物塑造专业指南

## 核心理念

**读者爱的是人，不是故事。**

一个好人物，能让平凡的故事发光。一个差人物，能让精彩的故事失色。

---

## 第一章：人设设计框架

### 三维人设法

#### 第一维：表面属性（读者第一眼看到的）

**基础信息**：
- 姓名、年龄、外貌
- 身份、职业、社会地位
- 穿着风格、口头禅

**作用**：建立第一印象，便于读者记忆

**示例**：
```
顾沉舟，28岁，顾氏集团总裁。
永远一丝不苟的西装，金丝边眼镜，
说话简洁，从不重复第二遍。
```

---

#### 第二维：隐藏属性（深入了解后发现的）

**性格深处**：
- 真实的恐惧和渴望
- 过去的创伤和阴影
- 不为人知的柔软一面

**作用**：制造反差萌，让人物立体

**示例**：
```
表面冷酷的顾沉舟，
其实怕黑（童年被关地下室），
私下会喂流浪猫，
收藏女主送的所有小东西。
```

---

#### 第三维：成长属性（故事发展中变化的）

**人物弧光**：
- 起点：故事开始时的状态
- 转折点：改变的关键事件
- 终点：故事结束时的状态

**作用**：展示人物成长，给读者满足感

**示例**：
```
起点：封闭内心，不相信爱情
转折点：女主一次次不离不弃
终点：学会信任，勇敢去爱
```

---

## 第二章：人物魅力来源

### 魅力的三个层次

#### 1. 独特性（让人记住）

**方法**：
- 给一个独特的标签
- 设计一个标志性的动作或习惯
- 创造独特的说话方式

**示例**：
```
标签：永远带着一把黑伞（不管晴天雨天）
动作：紧张时会转戒指
说话：每句话结尾加\"明白？\"
```

---

#### 2. 共鸣性（让人喜欢）

**方法**：
- 给人物一个小缺陷
- 展示脆弱的一面
- 有 relatable 的困境

**示例**：
```
缺陷：路痴，明明很近也会迷路
脆弱：深夜会失眠，一个人发呆
困境：努力工作但总是不被认可
```

---

#### 3. 成长性（让人投入）

**方法**：
- 人物有明显的变化
- 成长过程真实可信
- 成长结果令人满意

**示例**：
```
变化：从自卑到自信
过程：一次次失败但不放弃
结果：终于证明自己
```

---

### 缺陷设计原则

**好的缺陷**：
- 让人物更真实
- 能推动剧情
- 可以被克服或接纳

**示例**：
- 太骄傲 → 学会谦逊
- 太自卑 → 找到自信
- 太冲动 → 学会冷静
- 太封闭 → 学会打开心扉

**坏的缺陷**：
- 让人物不可爱（太坏、太蠢）
- 无法被原谅（伤害无辜）
- 和剧情无关

---

## 第三章：人物关系网

### 关系类型

#### 1. 爱情线（CP）
**核心**：化学反应

**设计要点**：
- 互补或相似（都有看点）
- 有共同的目标或困境
- 有必须在一起的理由
- 有阻碍他们在一起的力量

**关系动态**：
```
陌生 → 好奇 → 吸引 → 暧昧 → 确认 → 磨合 → 升华
```

---

#### 2. 亲情线
**核心**：羁绊与和解

**常见类型**：
- 严厉的父亲 → 理解与爱
- 控制欲强的母亲 → 边界与独立
- 失散的亲人 → 寻找与团聚
- 代沟 → 沟通与接纳

---

#### 3. 友情线
**核心**：陪伴与支持

**设计要点**：
- 闺蜜/兄弟要有区分度
- 不能只是工具人
- 可以有小的冲突和和解

---

#### 4. 敌对关系
**核心**：冲突与对抗

**反派设计**：
- 有合理的动机（不是单纯坏）
- 和主角有某种联系（镜像、过去、相似）
- 给主角足够的压力

---

### 关系网图示

```
        [主角]
       /  |  \
      /   |   \
   [CP] [闺蜜] [家人]
    |      |      |
   [情敌] [同事] [反派]
    |              |
   [助攻]        [手下]
```

**原则**：每个人物都要和主角有关系，关系要多样。

## 第四章：人物塑造技巧

### 展示 vs 讲述

❌ **讲述**（坏）：
```
她是一个很善良的人。
```

✅ **展示**（好）：
```
看到流浪猫在雨里发抖，
她脱下外套裹住它，
自己淋着雨跑回家。
```

**原则**：通过行动展示性格，而不是直接告诉读者。

---

### 对话塑造法

**每个人物要有独特的说话方式**：

| 人物 | 特征 | 示例 |
|------|------|------|
| 高冷男主 | 简短，命令式 | \"过来。\" \"不用。\" |
| 活泼女主 | 多语气词，跳跃 | \"真的吗！哇，太棒了！\" |
| 温柔男二 | 体贴，询问式 | \"你还好吗？\" \"需要帮忙吗？\" |
| 傲娇女主 | 口是心非 | \"谁在乎你啊！\"（其实在乎） |

---

### 细节塑造法

**用细节让人物鲜活**：

- **习惯性动作**：紧张时咬嘴唇、思考时转笔
- **特殊喜好**：只喝美式、讨厌香菜、喜欢雨天
- **小怪癖**：强迫症、收集癖、洁癖
- **身体语言**：眼神躲闪、抱臂、摸鼻子

---

## 第五章：主角 vs 配角

### 主角设计

**要求**：
- 有明确的目标和动机
- 有明显的成长弧线
- 有让读者喜欢的地方
- 有 relatable 的困境

**常见主角类型**：
| 类型 | 特点 | 示例 |
|------|------|------|
| 成长型 | 从弱到强 | 废柴逆袭 |
| 天才型 | 能力强但有缺陷 | 高冷学霸 |
| 平凡型 | 普通人做不普通的事 | 邻家女孩 |
| 反转型 | 表面和实际不同 | 隐藏身份 |

---

### 配角设计

**原则**：
- 每个配角都要服务于主线或主角成长
- 配角也要有自己的人生，不只是工具
- 配角之间要有区分度

**配角类型**：
| 类型 | 功能 | 示例 |
|------|------|------|
| 助攻 | 推动主角感情 | 闺蜜、兄弟 |
| 阻碍 | 制造冲突 | 情敌、反派 |
| 导师 | 引导主角成长 | 师父、前辈 |
| 喜剧 | 调节气氛 | 搞笑担当 |
| 悲剧 | 触动情感 | 牺牲的角色 |

# 爽点虐点设计：情绪操控指南

## 核心真理

**女频读者付费买的是情绪价值！**

- 爽点 = 让读者痛快、满足、解气 → **追更+付费**
- 虐点 = 让读者心疼、揪心、流泪 → **投入+粘性**
- 没有情绪起伏 = 平 → **流失**

**我们的目标：精准操控读者情绪，让她欲罢不能！**

---

## 第一章：女频10大黄金爽点

### 爽点1：打脸（最基础最爽）

**定义**：反派挑衅 → 主角反击 → 反派惨败

**公式**：
```
铺垫（反派嚣张）→ 升级（反派过分）→ 爆发（主角反击）→ 余波（众人震惊）
```

**关键**：
- 反派要够讨厌（让读者生气）
- 铺垫要够长（让读者着急）
- 反击要够狠（让读者解气）
- 围观群众要有反应（爽感放大）

**示例**：
```
绿茶：\"姐姐，你这件衣服是去年的款吧？\"
众人：窃笑
女主：\"是啊，不像妹妹，年年换新，
      毕竟......被包养的就是不一样。\"
（拿出照片：绿茶和老男人）
众人：震惊！
绿茶：脸色惨白
```

**频率**：每5-10章一次小打脸，每20-30章一次大打脸

---

### 爽点2：护短（女频最爱）

**定义**：男主/亲友无条件保护女主，为她出头

**类型**：
| 类型 | 示例 | 效果 |
|------|------|------|
| 霸气护妻 | \"我的人，你也敢动？\" | 安全感爆棚 |
| 温柔守护 | 默默为她解决所有问题 | 暖心感动 |
| 双标现场 | 对别人冷淡，对她百依百顺 | 专属感 |
| 打脸护短 | 当众为她撑腰，让反派难堪 | 爽+甜 |

**关键**：
- 女主必须先受委屈（让读者心疼）
- 护短要高调（让所有人知道）
- 护短后要撒糖（情绪补偿）

**示例**：
```
女配：\"林晚就是靠关系进来的！\"
男主（突然出现）：\"你说得对。\"
女配（得意）
男主：\"她靠的就是我。有意见？\"
众人：震惊！
女主：脸红
```

---

### 爽点3：逆袭（最励志）

**定义**：从低谷到巅峰，从被看不起到让人仰望

**公式**：
```
被轻视 → 隐忍努力 → 关键时刻 → 惊艳全场 → 众人后悔
```

**类型**：
- 能力逆袭：废物变天才
- 身份逆袭：灰姑娘变公主
- 颜值逆袭：丑小鸭变天鹅
- 事业逆袭：底层到顶层

**关键**：
- 低谷要够低（对比强烈）
- 过程要有努力（不能全靠金手指）
- 逆袭要够惊艳（全场震惊）
- 之前看不起的人要后悔（爽点核心）

---

### 爽点4：撒糖（女频刚需）

**定义**：男女主甜蜜互动，让读者露出姨母笑

**撒糖等级**：
| 等级 | 内容 | 频率 |
|------|------|------|
| 初级 | 眼神交流、关心话语 | 每章 |
| 中级 | 肢体接触、暧昧对话 | 每3章 |
| 高级 | 亲密接触、深情告白 | 每10章 |
| 顶级 | 正式在一起、同居、结婚 |  milestone |

**撒糖技巧**：
- 公开场合的偏爱（双标最甜）
- 私下里的温柔（反差萌）
- 吃醋（占有欲）
- 只有两个人知道的小秘密

---

### 爽点5：掉马（最刺激）

**定义**：隐藏身份被揭露，众人震惊

**类型**：
- 能力掉马：原来她是天才！
- 身份掉马：原来她是千金/大佬！
- 关系掉马：原来他们是夫妻/情侣！

**公式**：
```
隐藏 → 被质疑/欺负 → 关键时刻 → 掉马 → 众人震惊 → 反派后悔
```

**关键**：
- 隐藏要够深（读者知道，书中人不知道）
- 质疑要够狠（让读者着急）
- 掉马要够震撼（全场寂静）
- 反应要够爽（之前看不起的人打脸）

---

### 爽点6：虐渣（最解气）

**定义**：反派得到报应，恶有恶报

**类型**：
- 当场报应：立刻打脸
- 延迟报应：秋后算账
- 众叛亲离：失去一切
- 身败名裂：社会性死亡

**关键**：
- 反派要够坏（让读者恨）
- 报应要够惨（让读者解气）
- 要让反派后悔（但晚了）
- 不能圣母（主角不能原谅）

---

### 爽点7：被偏爱（最暖心）

**定义**：主角被特殊对待，成为唯一

**表现**：
- \"我只对你这样\"
- \"你是例外\"
- \"其他人不行，你可以\"
- \"我的都是你的\"

**关键**：
- 对比要明显（对别人vs对女主）
- 要持续（不是一次性）
- 要高调（让所有人知道女主特殊）

---

### 爽点8：升级（最满足）

**定义**：主角能力/地位/财富提升

**类型**：
- 能力提升：修炼升级、技能解锁
- 地位提升：升职、获得认可
- 财富提升：赚钱、暴富
- 关系升级：从朋友到恋人

**关键**：
- 升级要有过程（不能一蹴而就）
- 升级要有代价（努力/牺牲）
- 升级后要展示（打脸之前看不起的人）

---

### 爽点9：真相大白（最痛快）

**定义**：误会解除，真相揭露，坏人暴露

**公式**：
```
误会 → 被冤枉 → 证据出现 → 真相大白 → 坏人暴露 → 众人道歉
```

**关键**：
- 误会要够深（让读者憋屈）
- 证据要铁（无法反驳）
- 揭露要当众（让所有人知道）
- 道歉要诚恳（之前冤枉的人后悔）

---

### 爽点10：圆满结局（最终爽点）

**定义**：所有愿望实现，所有坏人得到报应，爱情圆满

**要素**：
- 事业成功
- 爱情圆满
- 仇人败落
- 亲友幸福
- 孩子（可选）

**关键**：
- 所有伏笔回收
- 所有承诺兑现
- 让读者感到\"这趟追更值了\"

---

## 第二章：虐点设计——危险的艺术

### 虐点的价值

**好的虐点**：
- 让读者心疼主角 → 情感投入
- 让感情更深刻 → 虐后更甜
- 让成长更真实 → 没有痛苦就没有成长

**坏的虐点**：
- 让读者愤怒（而不是心疼）→ 弃书
- 让主角太惨 → 压抑
- 为虐而虐 → 狗血

### 安全虐点清单（不会毒读者）

#### ✅ 安全虐点1：误会（可控）
- 有合理基础（不是强行误会）
- 会解除（读者知道是暂时的）
- 解除后有补偿（更甜）

#### ✅ 安全虐点2：外部阻碍（非主角错）
- 家族反对
- 身份差距
- 外界压力
- 这些都不是主角的错，读者不会生气

#### ✅ 安全虐点3：牺牲（有意义）
- 为对方牺牲
- 牺牲后有回报
- 读者感动而不是愤怒

#### ✅ 安全虐点4：成长痛（必要过程）
- 主角必须经历的挫折
- 为了变得更好
- 读者理解并支持

### 危险虐点清单（容易毒读者）

#### ❌ 危险虐点1：出轨/背叛
- 除非你能写得极好，否则不要碰
- 读者很难原谅
- 容易掉粉

#### ❌ 危险虐点2：强行降智
- 主角突然变蠢
- 读者会愤怒：\"这都不解释？\"

#### ❌ 危险虐点3：圣母/原谅
- 主角原谅伤害自己的人
- 读者会气死：\"这都能原谅？\"

#### ❌ 危险虐点4：重复虐
- 同样的虐点来三次以上
- 读者会疲劳：\"又来了？\"

## 第三章：爽虐比例——黄金公式

### 不同风格的配比

| 风格 | 爽:虐 | 适用类型 | 读者群体 |
|------|-------|----------|----------|
| 纯甜宠 | 9:1 | 现言、古言甜宠 | 想要放松的读者 |
| 轻虐甜 | 7:3 | 先虐后甜、破镜重圆 | 喜欢情绪波动的读者 |
| 甜虐交织 | 5:5 | 虐恋情深、权谋 | 喜欢深度的读者 |
| 虐主文 | 3:7 | 虐身虐心、BE美学 | 特定受众（小众） |

### 推荐配比（爆款向）：轻虐甜 7:3

**理由**：
- 够甜：满足追更动力
- 够虐：增加情感投入
- 安全：不容易毒读者

### 爽虐分布节奏

```
章1-5：爽（建立好感）
章6-10：小虐（增加投入）
章11-15：爽（补偿）
章16-20：中虐（情感升级）
章21-25：大爽（打脸/确认关系）
...
```

**原则**：
- 虐后必有爽（补偿）
- 大虐后必有大爽（情绪释放）
- 连续虐不超过3章（防流失）
- 连续爽不超过5章（防腻）

## 第五章：爽点设计实战

### 爽点设计流程

1. **确定位置**：这章/这段需要什么情绪？
2. **选择类型**：用哪种爽点？
3. **设计铺垫**：怎么让读者期待？
4. **设计爆发**：怎么让读者爽到？
5. **设计余波**：怎么让爽感延续？

### 爽点强度控制

| 强度 | 适用场景 | 示例 |
|------|----------|------|
| 小爽 | 日常调剂 | 小打脸、小撒糖 |
| 中爽 | 阶段性目标 | 中打脸、关系突破 |
| 大爽 | milestone | 大打脸、掉马、确认关系 |
| 超爽 | 大高潮 | 终极打脸、圆满结局 |

### 爽点密度

- **每章**：至少1个小爽点（撒糖/小互动）
- **每3章**：1个中爽点（打脸/护短）
- **每10章**：1个大爽点（掉马/关系突破）
- **每30章**：1个超爽点（大高潮）

---

## 第六章：虐点设计实战

### 虐点设计流程

1. **确定目的**：为什么要虐？（推进感情/人物成长/增加投入）
2. **选择类型**：用什么虐点？（误会/外部/牺牲）
3. **控制程度**：虐到什么程度？（心疼但不愤怒）
4. **设计解决**：怎么解决？（让读者有期待）
5. **设计补偿**：解决后怎么补偿？（更甜/更爽）

### 虐点安全线

**虐之前检查**：
- [ ] 这个虐点有必要吗？
- [ ] 读者会心疼还是愤怒？
- [ ] 能解决吗？什么时候解决？
- [ ] 解决后怎么补偿？

**如果有任何一个答案不满意，重新设计！**

# 网文钩子与节奏控制指南

## 核心目标

让读者**停不下来**，一章接一章地看下去。

## 第一章：章末钩子（The Hook）

### 什么是好钩子？

让读者**必须**点开下一章的东西。

### 钩子六大类型

#### 1. 悬念型钩子
**定义**：提出一个问题，不立即回答

**示例**：
- \"她打开那封信，看到落款时，整个人僵住了。\"
- \"他笑着说出那个名字，却不知道身后的人已经变了脸色。\"

**适用**：悬疑、权谋、反转剧情

**技巧**：
- 悬念要在本章铺垫
- 答案要在3章内揭晓（别吊太久）
- 悬念要和主线相关

---

#### 2. 情绪型钩子
**定义**：情绪达到顶点，戛然而止

**示例**：
- \"她看着他转身离开的背影，眼泪终于掉了下来。\"
- \"那一刻，她终于明白，原来一切都是假的。\"

**适用**：虐文、情感高潮、人物成长

**技巧**：
- 情绪要真实积累
- 停在情绪最浓时
- 下一章开始给缓解或反转

---

#### 3. 冲突型钩子
**定义**：冲突即将爆发或刚刚爆发

**示例**：
- \"门被猛地推开，她抬头，正对上他那双冰冷的眼睛。\"
- \"刀架在脖子上，她却笑了：'你不敢。'\"

**适用**：打脸、复仇、正面对决

**技巧**：
- 冲突要有铺垫
- 停在冲突最紧张时
- 结果要在下一章揭晓

---

#### 4. 期待型钩子
**定义**：让读者期待即将发生的事

**示例**：
- \"明天，就是决定命运的那场考试。\"
- \"三天后，他就要见到那个改变他一生的人。\"

**适用**：升级、比赛、重要事件

**技巧**：
- 预告要有吸引力
- 结果要配得上期待
- 可以适度超预期

---

#### 5. 反转型钩子
**定义**：揭示一个意外信息

**示例**：
- \"她以为他是来救她的，直到看到他手里的枪。\"
- \"那份DNA报告上，赫然写着两个字：匹配。\"

**适用**：悬疑、身世揭秘、身份反转

**技巧**：
- 反转要有伏笔
- 不能太突兀
- 反转后要解释清楚

---

#### 6. 亲密型钩子（女频专用）
**定义**：感情即将有突破或刚刚有突破

**示例**：
- \"他的脸越来越近，她闭上了眼睛...\"
- \"'做我女朋友。'他说。而她的心跳漏了一拍。\"

**适用**：甜宠、暧昧、情感推进

**技巧**：
- 停在关键时刻前
- 让读者心痒痒
- 下一章可以延迟满足

---

### 钩子设计检查清单

每章结尾问自己：

- [ ] 读者有理由点开下一章吗？
- [ ] 这个钩子3章内能解决吗？
- [ ] 钩子和主线相关吗？
- [ ] 同样的钩子类型连续用了吗？（要变化）

---

## 第二章：节奏控制（Pacing）

### 节奏的核心原则

**张弛有度，波浪推进**

### 节奏四象限

```
        快节奏
           │
    冲突   │   高潮
    对抗   │   爆发
    追逐   │   揭秘
           │
───────────┼───────────
           │
    铺垫   │   日常
    过渡   │   感情
    回忆   │   恢复
           │
        慢节奏
```

**使用原则**：
- 快节奏后必须有慢节奏缓冲
- 慢节奏中要有小高潮维持兴趣
- 大高潮前要层层铺垫

---

### 场景节奏控制

#### 快节奏场景（500-1500字）
**特征**：
- 短句为主
- 动作连续
- 对话简短
- 情绪激烈

**适用**：打斗、追逐、争吵、危机

**示例**：
```
他冲了上去。
她没来得及反应。
刀光一闪——
血溅了出来。
```

---

#### 慢节奏场景（1500-3000字）
**特征**：
- 长短句结合
- 心理描写多
- 细节丰富
- 情绪细腻

**适用**：感情戏、日常、回忆、铺垫

**示例**：
```
她坐在窗边，看着窗外的雨。
一滴，两滴，三滴。
就像她此刻的心情，
湿漉漉的，
怎么也晒不干。
```

---

### 章节节奏安排

#### 标准章节结构（3000-5000字）

```
【开头】承接上章钩子（10%）
    ↓
【发展】推进剧情/感情（60%）
    ↓
【小高潮】一个爽点或情绪点（20%）
    ↓
【钩子】章末悬念（10%）
```

#### 连续章节节奏示例

| 章节 | 节奏 | 内容 | 钩子类型 |
|------|------|------|----------|
| 第10章 | 快 | 冲突爆发 | 冲突型 |
| 第11章 | 慢 | 情绪余波+感情升温 | 亲密型 |
| 第12章 | 中 | 日常+小事件 | 悬念型 |
| 第13章 | 快 | 危机出现 | 悬念型 |
| 第14章 | 快 | 解决危机 | 情绪型 |
| 第15章 | 慢 | 恢复+感情突破 | 亲密型 |

---

## 第三章：爽点设计

### 什么是爽点？

让读者感到**痛快、满足、解气**的情节。

### 女频常见爽点

| 爽点类型 | 示例 | 效果 |
|----------|------|------|
| 打脸 | 绿茶被揭穿 | 解气 |
| 逆袭 | 废物变天才 | 励志 |
| 护短 | 男主为女主出头 | 安全感 |
| 撒糖 | 甜蜜互动 | 幸福感 |
| 反转 | 真相大白 | 惊喜 |
| 成长 | 女主变强 | 成就感 |

### 爽点分布原则

- **每3-5章一个小爽点**
- **每10-15章一个大爽点**
- **大爽点前要有压抑期**

### 爽点公式

**压抑 → 爆发 → 余波**

例：
- 压抑：女主被欺负（3章铺垫）
- 爆发：女主反击，打脸反派（1章高潮）
- 余波：众人反应，男主心疼（1章收尾）

---

## 第四章：阅读流畅性

### 过渡技巧

#### 时间过渡
```
三天后...
转眼间，春天来了。
那个夜晚之后，一切都变了。
```

#### 场景过渡
```
与此同时，在城市的另一端...
她不知道的是...
视线转回三天前...
```

#### 情绪过渡
```
如果说刚才她还心存侥幸，
那么现在，
她彻底绝望了。
```

### 避免阅读障碍

❌ **不要**：
- 大段背景介绍（分散插入）
- 过多人物同时出场（分批介绍）
- 复杂的时间线（保持清晰）
- 过长的段落（适当分段）

✅ **要**：
- 每章有明确的焦点
- 视角转换要清晰
- 信息分批透露
- 保持悬念和期待

# 网文人味写作助手

## 核心原则：保持\"人味\"

\"人味\"是指文字读起来像真人写的，有情感、有温度、有瑕疵、有生命力。避免AI腔、机械感和过度修饰。

### 人味的特征

1. **情感真实**：角色有真实的情绪反应，不是完美理性的
2. **语言自然**：对话口语化，符合人物身份和场景
3. **细节生动**：有具体的感官描写，不是抽象概括
4. **节奏变化**：长短句结合，有呼吸感
5. **不完美美**：允许适当的重复、口语化表达、甚至小瑕疵

## 写作指导框架

### 开篇阶段

**目标**：快速抓住读者，建立世界观和人物印象

**人味技巧**：
- 用具体场景开场，避免大段背景介绍
- 主角要有\"小缺陷\"或\"小执念\"，让读者产生共鸣
- 第一幕要有一个\"钩子\"——悬念、冲突或意外

**检查清单**：
- [ ] 前300字有没有让读者想知道\"然后呢\"？
- [ ] 主角的性格是否通过行动而非描述展现？
- [ ] 世界观是自然而然流露，还是生硬灌输？

### 发展阶段的\"人味\"维护

**情感线**：
- 角色成长要有\"反复\"，不是直线上升
- 感情发展要有\"暧昧期\"，不要进展太快
- 冲突要有\"灰色地带\"，避免非黑即白

**剧情线**：
- 每个事件后要有\"情绪余波\"
- 伏笔要\"藏\"得自然，不要刻意
- 高潮前要有\"压抑期\"，让爆发更有力

### 对话写作

**DO**：
- 用\" said/说 \"为主，少用花哨的说话动词
- 对话要\"带刺\"或\"带糖\"，有潜台词
- 每个人物有独特的口头禅或表达方式

**DON'T**：
- 用对话进行大段背景说明
- 让角色\"解释\"剧情给读者听
- 所有人说话风格都一样

### 描写技巧

**五感写作法**：
- 视觉：不只写颜色，写光影、质感
- 听觉：环境音、心跳声、呼吸声
- 嗅觉：往往能唤起最强烈的记忆
- 触觉：温度、质地、疼痛
- 味觉：苦涩、甜蜜、酸涩（常用来隐喻情绪）

**人味描写示例**：
```
AI腔：\"他的心情很悲伤。\"
人味：\"他盯着那杯已经凉透的咖啡，没喝一口。\"
```

## 网文特有技巧

### 节奏控制

- **章末钩子**：每章结尾留一个小悬念或情绪高潮
- **爽点分布**：每隔几章要有让读者\"爽\"到的情节
- **张弛有度**：紧张情节后要有轻松调剂

### 人物塑造

- **标签化+反标签**：先给读者一个印象，再打破它
- **缺陷魅力**：完美角色没人爱，有缺陷才真实
- **成长弧线**：主角要有可见的变化轨迹

### 情感张力

- **延迟满足**：想要的不要立刻给
- **误会与和解**：但要避免狗血，误会要有合理基础
- **牺牲与得到**：真正的情感需要付出代价

## 自检问题

写完一段后问自己：

1. **这段如果真人写，会这样写吗？**
2. **读者能\"感觉\"到情绪，还是只能\"知道\"情绪？**
3. **有没有哪个句子读起来像说明书？**
4. **如果念出来，会尴尬吗？**
5. **这段删掉，故事还成立吗？**（如果不成立，说明它是必要的）

## 常见AI腔 vs 人味对照

| AI腔 | 人味 |
|------|------|
| 他感到非常难过 | 他转过身，不想让人看见他的表情 |
| 她美丽动人 | 她笑起来的时候，左边脸颊有个酒窝 |
| 他们陷入了爱河 | 他开始注意到她喝咖啡时总要加两块糖 |
| 这是一个危险的计划 | \"你确定？\"他问，手指在桌面上敲出紧张的节拍 |
| 时间流逝 | 窗外的蝉鸣从聒噪变得稀疏 |

## 补充设定
1. 禁用词库（触发即违规）：
绝对禁止使用以下烂俗网文/AI高频词：瞬间、宛如、犹如、倒吸一口凉气、瞳孔骤缩、嘴角勾起一抹弧度/冷笑、不可置信、轰然炸开、仿佛、难以置信、恐怖如斯。
禁止使用排比句和过度对称的四字成语堆砌等等类似形容词。
2. 动词为王（Show, Don't Tell）：
删掉所有多余的形容词和副词。不要写‘他发出了极其恐怖的重击’，而是写‘他的拳峰砸碎了对方的下颌骨，碎骨刺穿了皮肉’。
战斗场面必须写出物理学逻辑和生理反应：肌肉的拉扯、骨骼的闷响、血液的温度、重心的偏移、汗水与腥味的混合。
3. 句式与节奏（长短句结合）：
日常描写用中长句，铺垫压抑的氛围。
战斗和高潮爆发时，必须使用极短句。（例如：“砰！血花四溅。他没有停顿，再出一拳。咔嚓。头骨碎裂。”）营造干脆利落、不服就干的极道爽感。
4. 对话粗粝化：
角色说话要符合其身份。末世蛮族和底层奴隶不会长篇大论，说话要短促、直接、带脏字或黑话。生死搏杀时绝对不要安排角色互相解说招式！
5. 视角解构的沉浸感：
贴脸描写。不要开上帝视角去写‘这幅画面太惨烈了’，而是通过主角的五感去写：他闻到了什么味道（铁锈般的血腥味、粪便味）、感受到了什么温度（冰冷的刀锋、滚烫的内脏）。

## 使用建议

当用户需要：
1. **润色章节** → 检查人味程度，提出具体修改建议
2. **构思情节** → 提供多个有人味的发展方向
3. **塑造人物** → 帮助设计有缺陷、有魅力的角色
4. **写对话** → 让对话更自然、有潜台词
5. **突破瓶颈** → 分析卡点，提供人味解决方案

永远记住：**让读者忘记这是文字，而是真实发生的故事**。

# 开篇黄金三章：生死之战

## 残酷真相

**女频读者3章定生死：**
- 第1章：看不进去 → 直接划走

- 第1章：有没有卖点

- 第3章：值不值得培养

**我们的目标：让读者3章之内必须追更！**

---

## 第一章：第一章——黄金300字

### 核心任务

**300字内必须完成：**
1. 抓住眼球（让读者停下来）
2. 建立好奇（让读者想知道）
3. 设定基调（让读者知道这是什么类型）

### 7大黄金开局模式

#### 模式1：危机开局（最保险）
**公式**：主角陷入危机 + 悬念

**示例**：
```
刀架在脖子上的时候，林晚还在想——
她到底是哪里露出了马脚？
三天前她穿越到这本书里，
成了那个活不过三章的炮灰女配。
而现在，正是第三章。
```

**适用**：穿越、重生、逆袭文
**效果**：紧张感+代入感+悬念

---

#### 模式2：反差开局（最吸睛）
**公式**：表面A + 实际B（巨大反差）

**示例**：
```
全京城都知道，丞相府的嫡女是个傻子。
直到那天宫宴上，
她一首《将进酒》惊艳四座，
顺便把挑衅她的公主怼得哑口无言。
众人这才发现——
这哪是傻子？分明是扮猪吃老虎！
```

**适用**：打脸、逆袭、马甲文
**效果**：爽感+期待感

---

#### 模式3：暧昧开局（女频专用）
**公式**：亲密接触 + 身份悬念

**示例**：
```
沈知意醒来的时候，发现自己在一个陌生男人怀里。
更可怕的是——
她身上只穿了一件衬衫，还是他的。
男人睁开眼，声音沙哑：\"醒了？\"
沈知意：\"......我们昨晚？\"
男人：\"你说呢？\"
沈知意看着满地的衣服，陷入了沉思。
她连这男人是谁都不知道！
```

**适用**：现言、甜宠、先婚后爱
**效果**：脸红心跳+强烈好奇

---

#### 模式4：冲突开局（最直接）
**公式**：直接冲突 + 主角态度

**示例**：
```
\"离婚吧。\"
顾沉舟把协议书扔在桌上，语气平淡得像在讨论天气。
林晚笑了，拿起协议书看都没看就签了字。
\"好啊，财产对半，孩子归我。\"
顾沉舟皱眉：\"你——\"
\"怎么？\"林晚站起身，居高临下地看着他，
\"顾总不会以为，我林晚离了你活不下去吧？\"
```

**适用**：虐恋、复仇、离婚逆袭
**效果**：情绪冲击+人设建立

---

#### 模式5：悬念开局（最勾人）
**公式**：神秘事件 + 主角卷入

**示例**：
```
那封信送到我手上的时候，
送信人已经死了。
我打开信封，里面只有一张照片——
照片上的女孩，和我长得一模一样。
而照片背面写着一行字：
\"不要相信任何人，包括你自己。\"
```

**适用**：悬疑、推理、灵异
**效果**：强烈好奇+紧张感

---

#### 模式6：日常开局（最温馨）
**公式**：有趣日常 + 小冲突

**示例**：
```
\"老板，来碗牛肉面，多加牛肉！\"
\"姑娘，你已经连续吃了一个月了，
  不腻吗？\"
\"不腻！\"我拍着胸脯，
\"只要能看到你儿子，吃一年都不腻！\"
老板儿子从后厨探出头，脸红了。
```

**适用**：甜宠、美食、种田文
**效果**：轻松愉快+CP感

---

#### 模式7：重生/穿越开局（最套路但有效）
**公式**：死亡/穿越瞬间 + 新身份认知

**示例**：
```
上一秒，我还在颁奖典礼上被吊灯砸死。
下一秒，我成了古代一个即将被赐死的妃子。
看着面前端着毒酒的太监，
我深吸一口气——
没事，我可是拿过奥斯卡影后的人。
演，就完事了。
```

**适用**：重生、穿越、宫斗
**效果**：代入感+金手指期待

### 第二章必须完成

### 核心任务

### 第三章必须完成

1. **展示故事潜力**

   - 有更多人物、更多冲突、更多悬念

2. **建立情感连接**
   - 让读者喜欢/同情/支持主角
   - 让读者期待主角的成功

3. **埋下追更钩子**
   - 一个必须追下去的理由
   - 可以是剧情悬念，也可以是感情期待

```

```

```
主角决定不再忍耐，要开始反击
主角遇到了改变命运的机会
```

## 开篇致命错误（绝对不能犯）

### ❌ 错误1：大段背景介绍
**症状**：第一章全是世界观、家族关系、前世今生
**后果**：读者直接划走
**解决**：背景分散插入，先上冲突

### ❌ 错误2：人物太多
**症状**：第一章出场10个人，读者记不住
**后果**：混乱，不知道谁是主角
**解决**：主角先出场，其他人分批介绍

### ❌ 错误3：太平淡
**症状**：第一章就是日常，没有冲突
**后果**：读者觉得无聊
**解决**：第一章必须有冲突/悬念/钩子

### ❌ 错误4：节奏太慢
**症状**：铺垫太多，迟迟不进正题
**后果**：读者失去耐心
**解决**：前3章必须有一个小高潮

### ❌ 错误5：主角不讨喜
**症状**：主角太弱/太蠢/太坏
**后果**：读者无法代入
**解决**：给主角一个让人喜欢的点

### ❌ 错误6：类型不清
**症状**：读者看完3章不知道这是什么类型
**后果**：无法精准推荐
**解决**：前3章明确展示类型特色

# 女频情感线设计指南

## 核心理念

女频小说的灵魂是**情感**。读者追的是\"他们什么时候在一起\"，不是\"他们怎么打败反派\"。

## 情感六阶段模型

### 第一阶段：相遇期（第1-10%）

**目标**：建立第一印象，埋下情感种子

**关键要素**：
- **特殊感**：为什么是他/她？要有\"非你不可\"的理由
- **反差萌**：表面印象与真实性格的反差
- **命运感**：巧合、宿命、或不得不纠缠的理由

**常见开局**：
| 类型 | 示例 | 适用 |
|------|------|------|
| 冤家路窄 | 初次见面就结梁子 | 轻松甜宠文 |
| 救命之恩 | 一方救另一方 | 古言、仙侠 |
| 契约关系 | 被迫绑定在一起 | 先婚后爱 |
| 久别重逢 | 过去有交集 | 虐恋、破镜重圆 |
| 身份错位 | 误会对方身份 | 悬疑、权谋 |

**自检**：读者看完相遇，是否想知道\"他们接下来会怎样\"？

---

### 第二阶段：暧昧期（第10%-40%）

**目标**：让读者心痒痒，欲罢不能

**这是女频最重要的阶段！**

**暧昧的核心技巧**：

#### 1. 近而不亲
- 身体距离很近，但心理距离还在试探
- 例：整理衣领、擦嘴角、扶腰、共撑一把伞

#### 2. 言外之意
- 说的话有双层含义
- 例：\"你这么关心我？\" \"谁关心你了...\"（口是心非）

#### 3. 独占欲初现
- 看到对方和别人亲近会不舒服
- 但自己还不承认这是吃醋

#### 4. 特殊对待
- 对别人一种态度，对他/她是另一种
- 例：对所有人都冷淡，唯独对他笑

#### 5. 意外接触
- 摔倒被接住、手指触碰、共用物品
- 每次接触后双方的心理活动

**暧昧期禁忌**：
- ❌ 进展太快（表白太早）
- ❌ 一方太主动，另一方太被动（要推拉）
- ❌ 没有竞争者或危机感

---

### 第三阶段：确认期（第40%-50%）

**目标**：确定关系，给读者一个\"爽点\"

**确认方式**：
| 方式 | 特点 | 效果 |
|------|------|------|
| 表白 | 直接说\"我喜欢你\" | 直接但要有新意 |
| 行动确认 | 吻、牵手、公开关系 | 更有冲击力 |
| 危机确认 | 危险时刻认清心意 | 情感爆发强 |
| 误会解除 | 解开多年心结后在一起 | 虐文常用 |

**确认后的变化**：
- 身份转变带来的新冲突
- 从\"追\"到\"守\"的心态变化
- 外部阻力开始显现

---

### 第四阶段：稳定期（第50%-70%）

**目标**：展示\"在一起后\"的甜蜜，同时积累危机

**内容重点**：
- 日常甜：让读者感到\"我也想要这样的爱情\"
- 小磨合：展示真实相处的细节
- 外部压力：家族、事业、情敌等阻力

**稳定期陷阱**：
- ❌ 太甜太腻（要有起伏）
- ❌ 太平淡（要持续有事件）
- ❌ 忘记主线（感情线要推动剧情）

---

### 第五阶段：危机期（第70%-90%）

**目标**：最大的考验，让感情升华

**危机类型**：
| 类型 | 示例 | 注意 |
|------|------|------|
| 误会型 | 以为对方背叛 | 不要太狗血，要有合理基础 |
| 牺牲型 | 为救对方付出巨大代价 | 虐身虐心但要有意义 |
| 选择型 | 爱情vs责任/亲情 | 两难选择最揪心 |
| 外部型 | 家族反对、生死危机 | 考验感情深度 |
| 成长型 | 一方变得强大，关系失衡 | 现实感强 |

**危机处理原则**：
- 危机要有铺垫，不能突然
- 双方都要付出，不能一方单方面牺牲
- 解决后感情要比之前更深

---

### 第六阶段：升华期（第90%-100%）

**目标**：圆满结局，让读者满足

**结局类型**：
- HE（Happy Ending）：在一起，幸福
- BE（Bad Ending）：因各种原因分开
- OE（Open Ending）：留有余味

**HE的层次**：
1. 在一起（基础）
2. 得到认可（家人、社会）
3. 共同成长（都变成更好的人）
4. 有未来（孩子、事业、生活愿景）

---

## 甜虐比例控制

### 不同风格的配比

| 风格 | 甜:虐 | 特点 |
|------|-------|------|
| 纯甜宠 | 9:1 | 全程高糖，虐点很少 |
| 轻虐甜 | 7:3 | 小虐怡情，大方向甜 |
| 虐恋情深 | 5:5 | 甜虐交织，刻骨铭心 |
| 虐文 | 3:7 | 虐为主，甜是点缀 |
| BE美学 | 2:8 | 越虐越美，结局遗憾 |

### 虐点设计原则

**好的虐点**：
- 虐是为了让感情更深
- 双方都在乎对方
- 虐后有补偿（糖）

**坏的虐点**：
- 为了虐而虐
- 一方伤害另一方太深（读者会脱粉）
- 虐后没有成长

# 大纲管理专业指南

## 核心理念

**大纲是地图，不是牢笼。**

好的大纲让你不迷路，但也允许你在路上发现新的风景。

---

## 第一章：大纲三层结构

### 第一层：故事核（1页）

**内容**：一句话概括你的故事

**公式**：
```
【主角】是一个【身份】的人，
TA想要【目标】，
但【阻碍】挡住了TA，
于是TA【行动】，
最终【结果】。
```

**示例**：
```
林晚是一个被家族抛弃的千金，
她想要夺回属于自己的一切，
但继母和妹妹处处陷害她，
于是她联手神秘总裁步步为营，
最终不仅复仇成功，还收获了真爱。
```

**作用**：
- 明确故事核心
-  pitch给编辑或读者
- 检查故事是否完整

---

### 第二层：主线大纲（5-10页）

**内容**：故事的主要脉络

**结构**：

```
【开端】（0-20%）
- 世界观介绍
- 主角登场
- 触发事件
- 主要人物出场

【发展】（20-60%）
- 主角行动
- 遭遇阻碍
- 结识盟友
- 感情发展
- 小高潮

【高潮】（60-90%）
- 最大危机
- 最终对决
- 真相揭露
- 情感爆发

【结局】（90-100%）
- 问题解决
- 人物归宿
- 主题升华
```

**每个部分包含**：
- 主要事件（发生了什么）
- 人物变化（主角的成长）
- 感情进展（CP的关系变化）
- 悬念设置（留下的钩子）

---

### 第三层：细纲/章纲（每章1段）

**内容**：每章的具体内容

**格式**：
```
第X章：章节标题
【场景】地点、时间
【人物】出场角色
【目标】这章要完成什么
【内容】具体情节
【钩子】章末悬念
```

**示例**：
```
第10章：雨夜的相遇
【场景】医院门口，雨夜
【人物】女主、男主
【目标】让男女主正式认识，建立第一印象
【内容】女主加班到深夜，下雨没带伞，
      在公交站躲雨。男主的车停在面前，
      让她上车。女主犹豫后上车，
      发现男主是白天面试她的总裁。
【钩子】男主说：\"我对你很感兴趣。\"
```

## 第二章：大纲撰写流程

### 步骤1：确定故事核（30分钟）

**问题清单**：
1. 主角是谁？有什么特别之处？
2. 主角想要什么？（目标）
3. 什么在阻止TA？（阻碍）
4. 谁会帮助/阻碍TA？（人物）
5. 故事的终点是什么？（结局）

**输出**：故事核一句话

---

### 步骤2：搭建主线框架（2-3小时）

**方法**：
1. 确定故事类型（甜宠/虐恋/复仇/成长）
2. 列出必须发生的大事件（10-15个）
3. 按时间顺序排列
4. 分配到大纲四部分

**工具**：便利贴法
- 每个事件写在便利贴上
- 在桌面/白板上排列
- 调整顺序直到流畅

---

### 步骤3：设计人物（2-3小时）

**主角设计**：
- 起点状态
- 终点状态
- 成长路径

**配角设计**：
- CP：和主角的关系发展
- 反派：动机和行动
- 助攻：功能和出场时机

---

### 步骤4：撰写细纲（持续进行）

**建议**：
- 先写前30章细纲
- 边写边补充后续
- 每完成10章，更新一次大纲

**细纲粒度**：
- 开头：详细（每章都要细纲）
- 中间：中等（每3-5章一个概括）
- 结尾：粗略（知道大致方向即可）

### 大纲更新原则

#### 什么时候更新大纲？

1. **写作前**：根据细纲写正文
2. **写作中**：发现更好的想法时
3. **每10章**：回顾并调整后续
4. **卡文时**：检查大纲是否合理
5. **大事件后**：确保后续连贯

#### 如何修改大纲？

**原则**：
- 小修改随时进行
- 大修改要慎重（影响后续）
- 记录修改原因
- 保留旧版本（对比）

**修改步骤**：
1. 在副本上修改
2. 检查连锁反应
3. 更新相关章节
4. 记录修改日志

---

### 伏笔管理

#### 伏笔记录表

| 伏笔 | 埋设章节 | 揭晓章节 | 状态 |
|------|----------|----------|------|
| 女主身世 | 第5章 | 第50章 | 已埋 |
| 男主旧伤 | 第12章 | 第45章 | 已埋 |
| 神秘人身份 | 第20章 | 第60章 | 待揭晓 |

**原则**：
- 埋设时就要想好揭晓时机
- 不要埋太多（读者记不住）
- 揭晓要有冲击力
- 遗忘的伏笔要及时回收或删除

## 第四章：常见问题解决

### 问题1：写不下去（卡文）

**诊断**：
- [ ] 细纲不够详细？
- [ ] 人物动机不明确？
- [ ] 情节逻辑有问题？
- [ ] 节奏太紧/太松？

**解决**：
1. 回到大纲，检查这章的目标
2. 和人物对话：TA现在最想要什么？
3. 跳过这章，写后面有灵感的部分
4. 增加一个小事件过渡

---

### 问题2：偏离大纲

**判断**：
- 偏离是更好的发展 → 更新大纲
- 偏离是跑题 → 拉回来
- 偏离是偷懒 → 坚持原大纲

**方法**：
1. 暂停写作
2. 对比原大纲和新想法
3. 评估哪个更好
4. 做出决定并执行

---

### 问题3：大纲太死板

**症状**：
- 严格按照大纲写，没有惊喜
- 人物像提线木偶
- 情节机械推进

**解决**：
1. 大纲只定方向，不定细节
2. 允许人物\"自己决定\"行动
3. 保留20%的即兴空间
4. 定期问自己：这是人物会做的吗？

---

### 问题4：没有大纲不敢写

**症状**：
- 过度规划，迟迟不动笔
- 害怕写得不够好
- 完美主义 paralysis

**解决**：
1. 先写故事核和开头10章细纲
2. 开始写，边写边完善
3. 接受第一稿不完美
4. 完成比完美重要
